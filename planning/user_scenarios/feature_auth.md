# 유저 시나리오: 사용자 인증 및 프로필

**기능**: 2.1 사용자 인증 및 프로필  
**작성자**: Athena PM  
**작성 일시**: 2024년  
**우선순위**: P0

---

## 시나리오 1: 신규 사용자 가입 (소셜 로그인)

### Given (전제 조건)
- 사용자가 FAN:STAGE 웹사이트에 처음 방문함
- 사용자는 Google/Kakao/Apple 계정을 보유하고 있음
- 사용자는 음악 공유 서비스를 사용하고 싶어함

### When (행동)
1. 사용자가 "시작하기" 버튼을 클릭함
2. 소셜 로그인 선택 화면에서 원하는 소셜 로그인을 선택함 (Google/Kakao/Apple)
3. 소셜 로그인 인증을 완료함
4. 닉네임 입력 화면에서 닉네임을 입력함 (필수)
5. 프로필 이미지를 선택함 (선택적)
6. "완료" 버튼을 클릭함

### Then (기대 결과)
- 사용자 계정이 생성됨
- 사용자가 메인 화면으로 이동함
- 사용자 프로필이 저장됨
- 다음 방문 시 자동 로그인됨

### 성공 기준
- 가입 완료까지 30초 이내
- 가입 전환율 70% 이상

---

## 시나리오 2: 기존 사용자 로그인

### Given (전제 조건)
- 사용자가 이전에 FAN:STAGE에 가입한 적이 있음
- 사용자는 로그아웃 상태임

### When (행동)
1. 사용자가 "로그인" 버튼을 클릭함
2. 이전에 사용한 소셜 로그인을 선택함
3. 소셜 로그인 인증을 완료함

### Then (기대 결과)
- 사용자가 자동으로 로그인됨
- 사용자가 메인 화면으로 이동함
- 사용자의 프로필 정보가 표시됨

---

## 시나리오 3: 프로필 수정

### Given (전제 조건)
- 사용자가 로그인되어 있음
- 사용자가 프로필 설정 페이지에 있음

### When (행동)
1. 사용자가 "프로필 수정" 버튼을 클릭함
2. 닉네임을 변경함
3. 프로필 이미지를 변경함
4. "저장" 버튼을 클릭함

### Then (기대 결과)
- 프로필 정보가 업데이트됨
- 변경된 정보가 즉시 반영됨
- 성공 메시지가 표시됨

---

## 엣지 케이스

### 엣지 케이스 1: 소셜 로그인 실패
**Given**: 사용자가 소셜 로그인을 시도함  
**When**: 소셜 로그인 인증이 실패함 (네트워크 오류, 권한 거부 등)  
**Then**: 
- 에러 메시지가 표시됨
- 다른 소셜 로그인 옵션을 선택할 수 있음
- 재시도 버튼이 제공됨

### 엣지 케이스 2: 중복 닉네임
**Given**: 사용자가 닉네임을 입력함  
**When**: 이미 사용 중인 닉네임을 입력함  
**Then**: 
- "이미 사용 중인 닉네임입니다" 메시지가 표시됨
- 다른 닉네임을 입력하도록 안내됨

### 엣지 케이스 3: 닉네임 미입력
**Given**: 사용자가 닉네임 입력 화면에 있음  
**When**: 닉네임을 입력하지 않고 "완료" 버튼을 클릭함  
**Then**: 
- "닉네임을 입력해주세요" 에러 메시지가 표시됨
- 닉네임 입력 필드에 포커스가 이동함

## 사용자 변화

**Before**: 음악 공유를 위해 여러 앱을 설치하고 계정을 만들어야 함  
**After**: 한 번의 로그인으로 모든 플랫폼과 연동 가능  
**변화 지표**: 가입 완료 시간 30초 이내, 가입 전환율 70% 이상
