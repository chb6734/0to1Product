# 사용자 시나리오 v2: 직관적이고 간단한 플로우

**작성자**: Athena PM (CEO, TPM 검토 반영)  
**작성 일시**: 2024년  
**기반**: PRD v4 (prd/prd_v4.md), CEO-TPM 검토

---

## 개요

이 문서는 CEO와 TPM의 검토를 반영하여 사용자 시나리오를 더 직관적이고 간단하게 재작성했습니다. 각 시나리오는 사용자의 목표를 먼저 명시하고, 핵심 플로우만 간단하게 작성했습니다.

---

## 핵심 원칙

1. **사용자 목표 우선**: 각 시나리오 시작에 "사용자가 원하는 것" 명시
2. **간결성**: 핵심 플로우만 명시, 선택적 단계는 별도 시나리오로 분리
3. **감성적 가치**: "왜 이 서비스를 사용하는가?" 강조
4. **번거로움 최소화**: 불필요한 단계 제거

---

## 1. 랜딩 페이지 시나리오

### 시나리오 1.1: 로그인 없이 서비스 체험하기

**사용자 목표**: 서비스를 먼저 체험해보고 싶다. 로그인하기 전에 어떤 서비스인지 확인하고 싶다.

**감성적 가치**: "로그인 없이도 바로 시작할 수 있어 부담이 없다"

**플로우:**
1. 랜딩 페이지 진입
2. "로그인 없이 체험하기" 버튼 클릭
3. 편지 생성 페이지로 이동 (데모 모드)
4. 곡 추가 및 메시지 작성
5. 완성 모달에서 링크 복사
6. 만족하면 "영구 저장하려면 로그인하기" 클릭하여 로그인

**성공 기준:**
- 데모 모드 진입 시간: 2초 이내
- 데모 모드에서 로그인 전환율: 30% 이상

---

### 시나리오 1.2: 친구에게 생일 축하 편지 보내기

**사용자 목표**: 친구 생일에 음악과 메시지를 함께 전달하고 싶다.

**감성적 가치**: "단순 링크가 아닌, 메시지와 함께 감성을 전달할 수 있다"

**플로우:**
1. 랜딩 페이지에서 "편지 만들기" 클릭
2. 로그인 (또는 데모 모드로 시작)
3. 곡 검색 및 추가 (생일 축하 곡들)
4. 메시지 작성 ("생일 축하해! 이 곡들 들으면서 행복한 하루 보내!")
5. 완성 모달에서 링크 복사
6. 친구에게 링크 공유

**성공 기준:**
- 편지 생성 완료 시간: 3분 이내
- 편지 공유율: 40% 이상

---

## 2. 데모 모드 시나리오

### 시나리오 2.1: 데모 모드로 편지 만들기

**사용자 목표**: 로그인 없이 편지를 만들어보고 싶다.

**감성적 가치**: "로그인 장벽 없이 바로 시작할 수 있다"

**플로우:**
1. 랜딩 페이지에서 "로그인 없이 체험하기" 클릭
2. 편지 생성 페이지 진입 (상단에 "데모 모드" 배지 표시)
3. 곡 추가 및 메시지 작성
4. "완료 및 공유" 클릭
5. 완성 모달에서 "임시 링크 (24시간 후 만료)" 확인
6. 링크 복사하여 공유 가능

**성공 기준:**
- 데모 모드 사용률: 50% 이상
- 데모 모드 편지 생성 완료율: 70% 이상

---

### 시나리오 2.2: 데모 모드에서 영구 저장하기

**사용자 목표**: 데모 모드로 만든 편지를 영구적으로 저장하고 싶다.

**감성적 가치**: "만든 편지를 잃어버리지 않고 보관할 수 있다"

**플로우:**
1. 데모 모드로 편지 생성 완료
2. 완성 모달에서 "영구 저장하려면 로그인하기" 클릭
3. 로그인 페이지로 이동
4. 소셜 로그인 완료
5. 온보딩 페이지에서 닉네임 입력 및 기본 플랫폼 설정
6. 보관함으로 이동 (데모 모드 편지가 자동으로 저장됨)

**성공 기준:**
- 데모 모드에서 로그인 전환율: 30% 이상
- 데이터 마이그레이션 성공률: 100%

---

## 3. 편지 생성 시나리오

### 시나리오 3.1: 기본 편지 생성 (간소화)

**사용자 목표**: 빠르게 편지를 만들고 싶다.

**감성적 가치**: "3분 안에 친구에게 음악을 선물할 수 있다"

**플로우:**
1. 편지 생성 페이지 진입
2. 곡 검색 및 추가 (3곡)
3. 메시지 작성
4. "완료 및 공유" 클릭
5. 완성 모달에서 링크 복사
6. 완료 (보관함으로 이동)

**개선점:**
- Step 1과 Step 2가 하나의 플로우로 통합됨
- 완성 모달로 플로우 중단 최소화

**성공 기준:**
- 편지 생성 완료 시간: 3분 이내
- 사용자 만족도: 4.0/5.0 이상

---

### 시나리오 3.2: 템플릿을 사용한 편지 생성

**사용자 목표**: 메시지 작성이 어려워서 템플릿을 사용하고 싶다.

**감성적 가치**: "메시지 작성 부담 없이 빠르게 편지를 만들 수 있다"

**플로우:**
1. 편지 생성 페이지 진입
2. 곡 추가
3. "템플릿 사용" 클릭
4. "생일 축하" 템플릿 선택
5. 메시지 자동 입력 확인
6. 메시지 수정 (선택적)
7. 완료

**참고**: P1 기능

---

## 4. 편지 확인 및 재생 시나리오

### 시나리오 4.1: 편지 확인 및 바로 재생하기

**사용자 목표**: 받은 편지를 확인하고 바로 음악을 듣고 싶다.

**감성적 가치**: "한 번의 클릭으로 바로 음악을 들을 수 있다"

**플로우:**
1. 보관함에서 편지 클릭
2. 편지 상세 페이지에서 메시지 읽기
3. "전체 재생" 버튼 클릭
4. 기본 플랫폼으로 자동 재생 (설정된 경우)
5. 또는 플랫폼 선택 모달에서 선택 후 재생

**개선점:**
- 기본 플랫폼이 설정되어 있으면 바로 재생
- 플랫폼 선택 단계 제거 (설정된 경우)

**성공 기준:**
- 편지 열람 후 재생 전환율: 60% 이상
- 재생까지 소요 시간: 5초 이내

---

### 시나리오 4.2: 편지 좋아요 및 공유하기

**사용자 목표**: 좋아하는 편지를 좋아요 표시하고 친구에게 공유하고 싶다.

**감성적 가치**: "좋아하는 음악을 쉽게 공유할 수 있다"

**플로우:**
1. 편지 상세 페이지에서 메시지 및 플레이리스트 확인
2. 좋아요 클릭
3. 공유 버튼 클릭
4. 공유 모달에서 "카카오톡 공유" 선택
5. 친구에게 공유

**성공 기준:**
- 편지 좋아요율: 30% 이상
- 편지 공유율: 40% 이상

---

## 5. 온보딩 시나리오

### 시나리오 5.1: 기본 플랫폼 설정하기

**사용자 목표**: 편지를 받을 때 자동으로 내 플랫폼에서 재생되게 하고 싶다.

**감성적 가치**: "매번 플랫폼을 선택하지 않아도 된다"

**플로우:**
1. 온보딩 페이지 진입
2. 닉네임 입력
3. "주로 사용하는 음악 플랫폼" 선택 (예: Spotify)
4. "시작하기" 클릭
5. 보관함으로 이동

**개선점:**
- 기본 플랫폼 설정이 온보딩에 포함됨
- 이후 편지 재생 시 자동으로 사용됨

**성공 기준:**
- 기본 플랫폼 설정률: 80% 이상
- 기본 플랫폼 사용률: 70% 이상

---

## 6. 통합 시나리오

### 시나리오 6.1: 전체 플로우 (신규 사용자, 개선)

**사용자 목표**: 처음 사용하는 사용자가 서비스를 체험하고 편지를 만들고 싶다.

**감성적 가치**: "로그인 없이도 바로 시작할 수 있고, 만족하면 저장할 수 있다"

**플로우:**
1. 랜딩 페이지에서 "로그인 없이 체험하기" 클릭
2. 데모 모드로 편지 생성 (곡 추가, 메시지 작성)
3. 완성 모달에서 링크 복사
4. 만족하여 "영구 저장하려면 로그인하기" 클릭
5. 로그인 완료
6. 온보딩에서 닉네임 입력 및 기본 플랫폼 설정
7. 보관함으로 이동 (편지가 자동으로 저장됨)

**개선점:**
- 로그인 장벽 제거
- 데모 모드로 먼저 체험
- 만족 시에만 로그인

**성공 기준:**
- 데모 모드 사용률: 50% 이상
- 데모 모드에서 로그인 전환율: 30% 이상
- 전체 가입 전환율: 70% 이상

---

### 시나리오 6.2: 편지 공유 및 수신 (개선)

**사용자 목표**: 편지를 공유하고 받은 사람이 쉽게 확인하고 재생할 수 있게 하고 싶다.

**감성적 가치**: "공유된 편지가 예쁘게 표시되고, 바로 재생할 수 있다"

**플로우:**
1. 사용자 A가 편지 생성 및 링크 복사
2. 사용자 B에게 링크 공유 (카카오톡)
3. 사용자 B가 링크 클릭
4. 편지 상세 페이지에서 예쁜 미리보기 확인 (OG 태그)
5. 메시지 읽기
6. "전체 재생" 클릭 (기본 플랫폼으로 자동 재생)
7. 좋아요 클릭

**개선점:**
- OG 태그로 공유 시 예쁜 미리보기 표시
- 기본 플랫폼으로 자동 재생

**성공 기준:**
- 공유된 편지 열람률: 30% 이상
- 공유된 편지 재생률: 60% 이상

## 7. 엣지 케이스 (간소화)

### 엣지 케이스 1: 데모 모드 데이터 손실 방지

**상황**: 데모 모드로 편지를 만들었는데 브라우저를 닫았음

**해결책**: 
- 로컬 스토리지에 자동 저장
- 다음 방문 시 "이전에 만든 편지가 있습니다" 메시지 표시
- 복구 옵션 제공

---

### 엣지 케이스 2: 기본 플랫폼 미설정 시

**상황**: 기본 플랫폼을 설정하지 않았는데 편지를 재생하고 싶음

**해결책**:
- "재생" 버튼 클릭 시 플랫폼 선택 모달 표시
- 선택한 플랫폼을 기본 플랫폼으로 저장할지 물어봄

---

### 엣지 케이스 3: 네트워크 오류

**상황**: 편지 생성 중 네트워크 연결이 끊김

**해결책**:
- 로컬 스토리지에 자동 저장
- "연결이 끊어졌습니다" 메시지 표시
- 재연결 시 자동 복구

---

## 8. 사용자 여정 요약

### 신규 사용자 여정 (개선)

```
랜딩 페이지
  ↓ [로그인 없이 체험하기]
데모 모드 편지 생성
  ↓ [만족]
로그인
  ↓ [온보딩]
보관함 (편지 자동 저장)
```

**개선점:**
- 로그인 장벽 제거
- 먼저 체험 후 로그인
- 전환율 향상 기대

---

### 기존 사용자 여정 (개선)

```
랜딩 페이지
  ↓ [편지 만들기]
편지 생성 (완성 모달)
  ↓ [완료]
보관함
  ↓ [편지 클릭]
편지 상세
  ↓ [전체 재생 - 기본 플랫폼으로 자동]
플랫폼에서 재생
```

**개선점:**
- 플로우 단순화
- 기본 플랫폼 자동 사용
- 번거로움 최소화
