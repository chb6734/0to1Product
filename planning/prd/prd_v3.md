## 2. 화면별 상세 명세

### 2.1 랜딩 페이지 (`/`)

#### 화면 목적
- 서비스의 핵심 가치 전달
- 사용자의 첫 액션 유도 (가입/편지 만들기)
- 서비스 신뢰도 구축

#### 화면 구성
1. **Header**
   - 로고 및 브랜드명
   - 네비게이션 (보관함, 둘러보기)
   - 로그인 버튼 또는 프로필

2. **히어로 섹션**
   - 배지: "플랫폼 중립적 음악 공유"
   - 헤딩: "어떤 플랫폼이든 음악을 공유하세요"
   - 본문: 서비스 설명
   - CTA 버튼: "편지 만들기", "데모 보기"

3. **추천 편지 섹션**
   - 섹션 제목: "추천 편지"
   - 편지 카드 그리드 (3개)
   - "전체 보기" 링크

4. **이용 방법 섹션**
   - 3단계 설명 (플레이리스트 만들기, 메시지 작성, 어디든 공유)

5. **CTA 섹션**
   - 헤딩 및 설명
   - "첫 편지 만들기" 버튼

#### 필수 동작 (P0)

**1. "편지 만들기" 버튼 클릭**
- **동작**: 로그인 상태 확인
  - 로그인 안 됨 → `/login`으로 이동
  - 로그인 됨 → `/create`로 이동
- **피드백**: 버튼 호버 시 opacity 변경

**2. 추천 편지 카드 클릭**
- **동작**: 해당 편지의 상세 페이지로 이동 (`/letters/[id]`)
- **피드백**: 카드 호버 시 약간의 scale 효과

**3. "전체 보기" 클릭**
- **동작**: `/discover`로 이동

**4. "첫 편지 만들기" CTA 클릭**
- **동작**: 로그인 상태 확인 후 `/login` 또는 `/create`로 이동

**5. "데모 보기" 버튼 클릭**
- **동작**: 데모 모달 표시 또는 데모 페이지로 이동 (P1)

#### 개선 동작 (P1)

**1. 데모 보기 기능**
- 데모 모달 또는 데모 페이지 제공
- 실제 편지 생성 프로세스 시연

**2. 스크롤 애니메이션**
- 섹션별 페이드인 애니메이션
- 스크롤 위치에 따른 등장 효과

**3. 추천 편지 동적 로딩**
- 실제 공개된 편지 데이터 표시
- 인기 편지 또는 최신 편지 우선 표시

**4. 소셜 증명**
- 사용자 수, 편지 수 등 통계 표시
- 사용자 후기 또는 추천사

#### 사용자 시나리오

**시나리오 1: 신규 사용자의 첫 방문**
1. 랜딩 페이지 진입
2. 히어로 섹션 스크롤하며 서비스 이해
3. "편지 만들기" 버튼 클릭
4. 로그인 페이지로 이동

**시나리오 2: 기존 사용자의 방문**
1. 랜딩 페이지 진입
2. 추천 편지 중 관심 있는 편지 클릭
3. 편지 상세 페이지로 이동

### 2.2 로그인 페이지 (`/login`)

#### 화면 목적
- 간편한 소셜 로그인 제공
- 신규 사용자 가입 및 기존 사용자 로그인

#### 화면 구성
1. **로고 아이콘**
   - FAN:STAGE 브랜드 아이콘

2. **헤딩**
   - "FAN:STAGE에 오신 것을 환영합니다"
   - "소셜 계정으로 간편하게 시작하세요"

3. **소셜 로그인 버튼**
   - Google 로그인
   - Kakao 로그인
   - Apple 로그인

4. **약관 동의 텍스트**
   - 이용약관 및 개인정보처리방침 링크

#### 필수 동작 (P0)

**1. 소셜 로그인 버튼 클릭**
- **동작**: 
  1. 로딩 스피너 표시
  2. 소셜 인증 팝업 열기
  3. 인증 완료 후 콜백 처리
- **성공 시**:
  - 신규 사용자 → `/onboarding`로 이동
  - 기존 사용자 → `/inbox`로 이동
- **실패 시**: 에러 메시지 표시

**2. 약관 링크 클릭**
- **동작**: 이용약관 또는 개인정보처리방침 페이지로 이동

#### 에러 처리

**1. 네트워크 오류**
- **메시지**: "연결에 실패했습니다. 다시 시도해주세요."
- **동작**: 재시도 버튼 표시

**2. 인증 거부**
- **메시지**: "로그인이 취소되었습니다."
- **동작**: 다시 로그인 버튼 클릭 가능

**3. 서버 오류**
- **메시지**: "일시적인 오류가 발생했습니다. 잠시 후 다시 시도해주세요."
- **동작**: 재시도 버튼 표시

#### 사용자 시나리오

**시나리오 1: 신규 사용자 가입**
1. 로그인 페이지 진입
2. Google 로그인 선택
3. Google 인증 완료
4. 온보딩 페이지로 이동

**시나리오 2: 기존 사용자 로그인**
1. 로그인 페이지 진입
2. 이전에 사용한 소셜 로그인 선택
3. 인증 완료
4. 보관함 페이지로 이동

**시나리오 3: 로그인 실패**
1. 로그인 버튼 클릭
2. 네트워크 오류 발생
3. 에러 메시지 확인
4. 재시도 또는 다른 소셜 로그인 선택

### 2.3 온보딩 페이지 (`/onboarding`)

#### 화면 목적
- 신규 사용자의 프로필 설정
- 서비스 사용을 위한 최소 정보 수집

#### 화면 구성
1. **헤딩**
   - "프로필 설정"
   - "닉네임과 프로필 사진을 설정해주세요"

2. **프로필 이미지 업로드**
   - 원형 프로필 이미지 영역
   - 업로드 버튼 (+ 아이콘)
   - "프로필 사진 (선택)" 텍스트

3. **닉네임 입력**
   - 입력 필드
   - 문자 수 표시 (0/20)
   - 필수 표시 (*)

4. **시작하기 버튼**
   - 비활성화 상태 (닉네임 미입력 시)

#### 필수 동작 (P0)

**1. 닉네임 입력**
- **동작**: 
  - 실시간 문자 수 표시
  - 최대 20자 제한
  - 유효성 검사 (한글/영문/숫자만 허용, 최소 2자)
- **피드백**: 
  - 유효하지 않은 문자 입력 시 경고 표시
  - 중복 검사 (실시간 또는 포커스 아웃 시)

**2. 프로필 이미지 업로드**
- **동작**: 
  1. 파일 선택 다이얼로그 열기
  2. 이미지 파일 선택 (JPG/PNG/GIF, 최대 5MB)
  3. 이미지 미리보기 표시
  4. 업로드 진행 상태 표시
- **피드백**: 업로드 성공 시 미리보기 업데이트

**3. 시작하기 버튼 클릭**
- **전제 조건**: 닉네임이 유효하고 중복되지 않음
- **동작**: 
  1. 프로필 정보 저장
  2. 로딩 표시
  3. 저장 완료 후 `/inbox`로 이동

#### 검증 규칙

**닉네임:**
- 길이: 2-20자
- 허용 문자: 한글, 영문, 숫자
- 금지: 특수문자, 공백
- 중복 불가

**프로필 이미지:**
- 형식: JPG, PNG, GIF
- 크기: 최대 5MB
- 비율: 1:1 권장 (원형 표시)

#### 사용자 시나리오

**시나리오 1: 기본 프로필로 시작**
1. 온보딩 페이지 진입
2. 닉네임 입력 ("김서연")
3. 프로필 이미지 건너뛰기
4. "시작하기" 클릭
5. 보관함 페이지로 이동

**시나리오 2: 프로필 이미지 포함**
1. 온보딩 페이지 진입
2. 프로필 이미지 업로드
3. 닉네임 입력
4. "시작하기" 클릭
5. 보관함 페이지로 이동

**시나리오 3: 닉네임 중복**
1. 온보딩 페이지 진입
2. 닉네임 입력 ("김서연")
3. 중복 검사 실패
4. "이미 사용 중인 닉네임입니다" 메시지 표시
5. 다른 닉네임 입력

### 2.4 편지 생성 페이지 (`/create`)

#### 화면 목적
- 음악 편지 생성
- 플레이리스트 구성 및 메시지 작성
- 편지 공유 링크 생성

#### 화면 구성

**Step 1: 편지 작성**

1. **헤더**
   - "음악 편지 만들기"
   - "곡을 추가하고 메시지를 작성하세요"

2. **왼쪽: 입력 폼**
   - 곡 추가 카드
     - 검색 입력 필드
     - 검색 결과 목록
   - 메시지 카드
     - 텍스트 영역 (최대 500자)
     - 문자 수 표시
     - 템플릿 사용 버튼
   - 공개 설정 카드
     - 비공개 체크박스
     - 설명 텍스트
   - 다음 버튼

3. **오른쪽: 미리보기**
   - 편지 미리보기 카드
   - 실시간 업데이트

**Step 2: 완성 화면**

1. **헤더**
   - "편지가 완성되었습니다!"
   - "링크나 QR코드로 공유하세요"

2. **공유 링크 카드**
   - 링크 입력 필드
   - 복사 버튼

3. **QR 코드 카드**
   - QR 코드 이미지
   - 설명 텍스트

4. **전체 공개 카드** (공개 설정 시)
   - 공개 상태 표시

5. **액션 버튼**
   - 수정하기 버튼
   - 완료 버튼

#### 필수 동작 (P0)

**Step 1 동작:**

**1. 곡 검색**
- **동작**: 
  1. 검색어 입력
  2. 자동완성 결과 표시 (최대 10개)
  3. 곡 선택 시 플레이리스트에 추가
- **피드백**: 검색 결과 호버 효과

**2. 곡 추가/삭제**
- **추가**: 검색 결과에서 곡 선택
- **삭제**: 곡 목록에서 삭제 버튼 클릭
- **피드백**: 플레이리스트 실시간 업데이트

**3. 곡 순서 변경**
- **동작**: 드래그 앤 드롭으로 순서 변경
- **피드백**: 드래그 중 시각적 피드백

**4. 메시지 작성**
- **동작**: 텍스트 입력 및 실시간 미리보기 업데이트
- **제한**: 최대 500자
- **피드백**: 문자 수 표시

**5. 템플릿 사용**
- **동작**: 템플릿 선택 시 메시지 자동 입력
- **템플릿 종류**: 생일, 위로, 응원, 고마움 등

**6. 공개 설정 변경**
- **동작**: 체크박스 토글
- **비공개**: 링크만 공유 가능
- **공개**: 둘러보기 페이지에 표시

**7. 다음 버튼 클릭**
- **전제 조건**: 메시지가 입력됨
- **동작**: Step 2로 이동

**Step 2 동작:**

**1. 링크 복사**
- **동작**: 클립보드에 복사 및 토스트 메시지 표시
- **피드백**: "링크가 복사되었습니다" 메시지

**2. QR 코드 표시**
- **동작**: 편지 링크를 포함한 QR 코드 생성 및 표시
- **용도**: 모바일에서 스캔하여 편지 열기

**3. 수정하기 버튼 클릭**
- **동작**: Step 1로 돌아가기 (입력 내용 유지)

**4. 완료 버튼 클릭**
- **동작**: 
  1. 편지 저장
  2. 로딩 표시
  3. 저장 완료 후 `/inbox`로 이동

#### 개선 동작 (P1)

**1. 임시 저장**
- 자동 저장 또는 수동 저장 버튼
- 브라우저 종료 시에도 데이터 보존

**2. 진행 표시기**
- Step 1/2 표시
- 진행률 표시

**3. 곡 재생 미리보기**
- 검색 결과에서 곡 미리듣기
- YouTube API 활용

#### 사용자 시나리오

**시나리오 1: 기본 편지 생성**
1. 편지 생성 페이지 진입
2. 곡 검색 및 추가 (3곡)
3. 메시지 작성
4. 공개 설정 (비공개)
5. "다음" 클릭
6. 링크 복사
7. "완료" 클릭
8. 보관함으로 이동

**시나리오 2: 템플릿 사용**
1. 편지 생성 페이지 진입
2. 곡 추가
3. "템플릿 사용" 클릭
4. "생일 축하" 템플릿 선택
5. 메시지 자동 입력
6. 메시지 수정
7. "다음" 클릭
8. 완료

**시나리오 3: 편지 수정**
1. Step 2 화면에서 "수정하기" 클릭
2. Step 1로 돌아가기
3. 곡 추가/삭제
4. 메시지 수정
5. "다음" 클릭
6. 완료

### 2.5 보관함 페이지 (`/inbox`)

#### 화면 목적
- 받은 편지 및 보낸 편지 관리
- 편지 목록 확인 및 접근

#### 화면 구성
1. **헤더**
   - "내 보관함"

2. **탭**
   - 받은 편지 탭
   - 보낸 편지 탭

3. **편지 목록**
   - 편지 카드 그리드 레이아웃 (2열)
   - 각 카드에 표시:
     - 보낸 사람/받는 사람 정보
     - 메시지 미리보기
     - 곡 수, 재생 수, 좋아요 수
     - 날짜

4. **빈 상태**
   - 아이콘 및 메시지
   - "편지 만들기" CTA

#### 필수 동작 (P0)

**1. 탭 전환**
- **동작**: 받은 편지/보낸 편지 목록 변경
- **피드백**: 활성 탭 하이라이트

**2. 편지 카드 클릭**
- **동작**: 해당 편지의 상세 페이지로 이동 (`/letters/[id]`)
- **피드백**: 카드 호버 효과

**3. 빈 상태 표시**
- **조건**: 해당 탭에 편지가 없을 때
- **표시**: 아이콘, 메시지, "편지 만들기" 버튼

#### 개선 동작 (P1)

**1. 정렬 옵션**
- 최신순, 인기순, 곡 수순
- 드롭다운 메뉴로 선택

**2. 검색 기능**
- 보낸 사람/받는 사람 이름 검색
- 메시지 내용 검색
- 검색 결과 하이라이트

**3. 무한 스크롤**
- 스크롤 시 추가 편지 로드
- 로딩 인디케이터 표시

**4. 편지 삭제**
- 편지 카드에 삭제 버튼 추가
- 확인 모달 표시 후 삭제

**5. 편지 즐겨찾기**
- 즐겨찾기 버튼 추가
- 즐겨찾기한 편지 별도 탭 또는 필터

#### 사용자 시나리오

**시나리오 1: 받은 편지 확인**
1. 보관함 페이지 진입
2. "받은 편지" 탭 확인 (기본)
3. 편지 목록 확인
4. 관심 있는 편지 클릭
5. 편지 상세 페이지로 이동

**시나리오 2: 보낸 편지 확인**
1. 보관함 페이지 진입
2. "보낸 편지" 탭 클릭
3. 보낸 편지 목록 확인
4. 편지 클릭하여 상세 확인

**시나리오 3: 빈 상태**
1. 보관함 페이지 진입
2. 받은 편지가 없음
3. 빈 상태 메시지 확인
4. "편지 만들기" 버튼 클릭
5. 편지 생성 페이지로 이동

### 2.6 둘러보기 페이지 (`/discover`)

#### 화면 목적
- 공개된 편지 탐색
- 새로운 음악 발견
- 커뮤니티 참여

#### 화면 구성
1. **헤더**
   - "둘러보기"
   - "다른 사람들이 공유한 음악 편지를 둘러보세요"

2. **편지 목록**
   - 편지 카드 그리드 레이아웃 (2열)
   - 각 카드에 표시:
     - 보낸 사람 정보
     - 제목 또는 메시지 미리보기
     - 곡 수, 재생 수, 좋아요 수
     - 날짜

#### 필수 동작 (P0)

**1. 편지 카드 클릭**
- **동작**: 해당 편지의 상세 페이지로 이동 (`/letters/[id]`)
- **피드백**: 카드 호버 효과

**2. 스크롤**
- **동작**: 추가 편지 로드 (무한 스크롤 또는 페이지네이션)
- **피드백**: 로딩 인디케이터 표시

#### 개선 동작 (P1)

**1. 카테고리 필터**
- 장르별 필터 (팝, 록, 힙합 등)
- 분위기별 필터 (신나는, 감성적인 등)
- 필터 선택 시 목록 업데이트

**2. 검색 기능**
- 편지 제목/메시지 검색
- 보낸 사람 검색
- 검색 결과 하이라이트

**3. 정렬 옵션**
- 인기순 (좋아요 수 기준)
- 최신순 (생성일 기준)
- 재생 수순

**4. 추천 편지 섹션**
- 사용자 취향 기반 추천
- 트렌딩 편지 섹션

#### 사용자 시나리오

**시나리오 1: 편지 탐색**
1. 둘러보기 페이지 진입
2. 편지 목록 스크롤
3. 관심 있는 편지 클릭
4. 편지 상세 페이지로 이동

**시나리오 2: 카테고리 필터**
1. 둘러보기 페이지 진입
2. "팝" 카테고리 선택
3. 팝 음악 편지만 표시
4. 편지 클릭하여 확인

### 2.7 편지 상세 페이지 (`/letters/[id]`)

#### 화면 목적
- 편지 전체 내용 표시
- 플레이리스트 확인 및 재생
- 편지 상호작용 (좋아요, 공유)

#### 화면 구성
1. **뒤로 가기 버튼**
   - 이전 페이지로 이동

2. **편지 카드**
   - 보낸 사람 정보 (프로필 이미지, 이름, 날짜)
   - 좋아요 버튼
   - 메시지 영역
   - 통계 (곡 수, 재생 수, 좋아요 수)

3. **플레이리스트**
   - 제목: "플레이리스트"
   - 전체 재생 버튼
   - 곡 목록 (번호, 앨범 아트, 제목, 아티스트, 더보기 버튼)

#### 필수 동작 (P0)

**1. 뒤로 가기**
- **동작**: 이전 페이지로 이동
- **피드백**: 버튼 호버 효과

**2. 좋아요 클릭**
- **동작**: 
  1. 좋아요 토글
  2. 좋아요 수 업데이트
  3. 하트 아이콘 상태 변경 (빈 하트 ↔ 채워진 하트)
- **피드백**: 애니메이션 효과

**3. 전체 재생 버튼 클릭**
- **동작**: 플레이리스트 전체 재생 (플랫폼 연동)
- **피드백**: 로딩 표시

**4. 곡 클릭**
- **동작**: 해당 플랫폼에서 곡 재생
- **표시**: 플랫폼별 링크 (Spotify, Apple Music, YouTube Music)

**5. 곡 더보기 버튼 클릭**
- **동작**: 메뉴 표시 (공유, 플랫폼별 링크)

#### 개선 동작 (P1)

**1. 공유 버튼**
- 링크 복사
- SNS 공유 (카카오톡, 페이스북, 트위터 등)
- 공유 모달 표시

**2. 곡 재생 미리보기**
- YouTube API를 활용한 미리듣기
- 곡별 재생 버튼

**3. 댓글 기능**
- 댓글 작성 및 표시
- 댓글 좋아요

**4. 편지 신고**
- 신고 버튼 추가
- 신고 사유 선택 모달

**5. 편지 삭제** (본인이 보낸 편지인 경우)
- 삭제 버튼 표시
- 확인 모달 후 삭제

#### 사용자 시나리오

**시나리오 1: 편지 확인 및 좋아요**
1. 편지 상세 페이지 진입
2. 메시지 읽기
3. 플레이리스트 확인
4. 좋아요 클릭
5. 좋아요 수 증가 확인

**시나리오 2: 플레이리스트 재생**
1. 편지 상세 페이지 진입
2. "전체 재생" 버튼 클릭
3. 플랫폼 선택 (Spotify)
4. Spotify에서 플레이리스트 재생

**시나리오 3: 곡 개별 재생**
1. 편지 상세 페이지 진입
2. 특정 곡 클릭
3. 플랫폼별 링크 확인
4. 원하는 플랫폼에서 재생
