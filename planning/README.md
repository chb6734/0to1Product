# 기획 문서 모음

FAN:STAGE 프로젝트의 모든 기획 관련 문서를 모아둔 폴더입니다.

## 폴더 구조

```
planning/
├── idea.md                    # 최초 창업 아이디어 및 비전 문서
├── prd/                       # 제품 요구사항 문서 (PRD)
│   ├── prd_final.md          # 최종 PRD
│   ├── prd_v1.md             # PRD v1
│   ├── prd_v2.md             # PRD v2
│   └── execution_plan_v1.md  # 실행 계획
├── user_scenarios/            # 사용자 시나리오 문서
│   ├── feature_auth.md
│   ├── feature_letter_creation.md
│   ├── feature_letter_send_receive.md
│   └── feature_platform_integration.md
└── reviews/                   # CEO/PM 검토 및 피드백 문서
    ├── ceo_review_1.md       # CEO 초기 아이디어 검토
    ├── ceo_prd_review_1.md    # CEO PRD v1 검토
    ├── ceo_prd_review_2.md   # CEO PRD v2 검토
    └── pm_feedback_response_1.md # PM 피드백 응답
```

## 문서 읽기 순서

1. **아이디어 이해**: `idea.md` - 프로젝트의 비전과 목표
2. **제품 요구사항**: `prd/prd_final.md` - 최종 PRD 확인
3. **실행 계획**: `prd/execution_plan_v1.md` - 실행 전략
4. **사용자 시나리오**: `user_scenarios/` - 기능별 사용자 시나리오
5. **검토 과정**: `reviews/` - CEO/PM 간의 검토 및 피드백

## 주요 문서 설명

### idea.md
프로젝트의 최초 아이디어와 비전을 담은 문서입니다.
- 트렌드 반영한 사업 아이디어
- 문제 해결형 사업 아이디어
- 틈새 시장 발굴
- 성장 전략
- 기술 트렌드 활용

### prd/
제품 요구사항 문서(PRD)와 실행 계획을 포함합니다.
- **prd_final.md**: CEO 검토를 거쳐 최종 승인된 PRD
- **execution_plan_v1.md**: MVP 실행 계획 및 마일스톤

### user_scenarios/
각 기능별 상세한 사용자 시나리오를 정의합니다.
- Given-When-Then 형식으로 작성
- 엣지 케이스 포함
- 성공 기준 명시

### reviews/
CEO와 PM 간의 검토 및 피드백 과정을 기록합니다.
- CEO의 First Principles 기반 검토
- PM의 피드백 응답 및 개선 사항
- 승인 과정 및 의사결정
