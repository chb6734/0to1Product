# 🎨 반응형 디자인 전략

**작성자**: Maya Designer  
**작성 일시**: 2024년  
**기반**: 인터랙션 가이드 (design/interaction_guide.md), 디자인 시스템 (design/design_system.md)

---

## 반응형 디자인 원칙

1. **Mobile First**: 모바일을 우선 설계하고 데스크톱으로 확장
2. **Progressive Enhancement**: 기본 기능부터 시작하여 향상
3. **Touch First**: 터치 인터랙션을 우선 고려
4. **Content Priority**: 중요한 콘텐츠를 우선 표시
5. **Performance**: 모바일 성능 최적화

---

## 1. 브레이크포인트 정의

### 브레이크포인트 시스템

**Mobile**:
- **범위**: < 768px
- **특징**: 세로 레이아웃, 터치 최적화
- **타겟**: 스마트폰

**Tablet**:
- **범위**: 768px - 1024px
- **특징**: 가로/세로 레이아웃, 터치 + 마우스
- **타겟**: 태블릿

**Desktop**:
- **범위**: > 1024px
- **특징**: 가로 레이아웃, 마우스 최적화
- **타겟**: 데스크톱, 랩톱

### Tailwind CSS 브레이크포인트

```javascript
// tailwind.config.js
module.exports = {
  theme: {
    screens: {
      'sm': '640px',   // Small devices
      'md': '768px',   // Medium devices (Tablet)
      'lg': '1024px',  // Large devices (Desktop)
      'xl': '1280px',  // Extra large devices
      '2xl': '1536px', // 2X large devices
    },
  },
}
```

## 2. 모바일 우선 설계

### 터치 인터랙션 최적화

#### 터치 타겟 크기

**최소 크기**: 44px × 44px (Apple HIG 기준)

**권장 크기**:
- **버튼**: 44px - 56px 높이
- **아이콘 버튼**: 44px × 44px
- **링크**: 최소 44px 높이
- **입력 필드**: 44px 높이

**간격**:
- **터치 타겟 간격**: 최소 8px
- **권장 간격**: 12px - 16px

---

#### 터치 제스처

**탭**:
- **사용**: 기본 인터랙션
- **피드백**: Active 상태 (150ms)
- **영역**: 전체 터치 타겟

**롱 프레스**:
- **사용**: 컨텍스트 메뉴, 드래그 시작
- **지속 시간**: 500ms
- **피드백**: 햅틱 피드백 (iOS), 진동 (Android)

**스와이프**:
- **사용**: 삭제, 탭 전환
- **임계값**: 30% - 50% 스와이프
- **피드백**: 실시간 위치 표시

**핀치 투 줌**:
- **사용**: 이미지 확대
- **범위**: 1x - 3x
- **피드백**: 실시간 확대/축소

---

### 모바일 네비게이션

#### 하단 네비게이션

**구조**:
- **위치**: 하단 고정
- **높이**: 64px
- **항목**: 4-5개 (홈, 보관함, 생성, 프로필)
- **스타일**: 아이콘 + 라벨

**상태**:
- **활성**: Primary 500 컬러
- **비활성**: Gray 500 컬러
- **배경**: White (다크 모드: Gray 900)

---

#### 햄버거 메뉴

**사용**: 상단 네비게이션 (모바일)

**구조**:
- **위치**: 왼쪽 상단
- **아이콘**: 햄버거 아이콘 (3줄)
- **드로어**: 왼쪽에서 슬라이드

**드로어**:
- **너비**: 280px
- **배경**: White (다크 모드: Gray 900)
- **애니메이션**: Slide in from left (300ms)
- **오버레이**: 반투명 배경

---

### 모바일 레이아웃 패턴

#### 단일 컬럼 레이아웃

**사용**: 대부분의 모바일 화면

**특징**:
- **너비**: 전체 너비
- **패딩**: 좌우 24px
- **간격**: 세로 16px - 24px

**예시**:
- 편지 목록
- 편지 상세
- 편지 생성 (편집 영역)

---

#### 탭 레이아웃

**사용**: 여러 뷰 전환

**특징**:
- **탭**: 상단 고정
- **콘텐츠**: 탭 아래 스크롤
- **전환**: 스와이프 제스처 지원

**예시**:
- 편지 생성 (편집 / 미리보기)
- 보관함 (받은 편지 / 보낸 편지)

## 3. 레이아웃 적응

### 그리드 시스템

#### 모바일 (< 768px)

**컬럼**: 1 컬럼

**간격**: 16px

**패딩**: 좌우 24px

**예시**:
```css
.grid-mobile {
  display: grid;
  grid-template-columns: 1fr;
  gap: 16px;
  padding: 0 24px;
}
```

---

#### 태블릿 (768px - 1024px)

**컬럼**: 2 컬럼

**간격**: 24px

**패딩**: 좌우 32px

**예시**:
```css
@media (min-width: 768px) {
  .grid-tablet {
    grid-template-columns: repeat(2, 1fr);
    gap: 24px;
    padding: 0 32px;
  }
}
```

---

#### 데스크톱 (> 1024px)

**컬럼**: 3-4 컬럼

**간격**: 24px

**패딩**: 좌우 32px

**최대 너비**: 1280px, 중앙 정렬

**예시**:
```css
@media (min-width: 1024px) {
  .grid-desktop {
    grid-template-columns: repeat(3, 1fr);
    gap: 24px;
    max-width: 1280px;
    margin: 0 auto;
  }
}
```

### 콘텐츠 우선순위 조정

#### 모바일

**우선순위**:
1. 주요 CTA 버튼
2. 핵심 콘텐츠
3. 보조 정보
4. 푸터

**숨김 요소**:
- 보조 네비게이션
- 상세 설명 (접기/펼치기)
- 통계 정보 (접기/펼치기)

---

#### 데스크톱

**우선순위**:
1. 전체 네비게이션
2. 주요 콘텐츠 + 보조 콘텐츠
3. 사이드바 (선택)
4. 푸터

**표시 요소**:
- 전체 메뉴
- 상세 설명
- 통계 정보
- 관련 콘텐츠

---

### 네비게이션 패턴 변경

#### 모바일

**패턴**: 하단 네비게이션 + 햄버거 메뉴

**구조**:
- 상단: 로고, 햄버거 메뉴, 프로필
- 하단: 주요 액션 (홈, 보관함, 생성, 프로필)

---

#### 데스크톱

**패턴**: 상단 네비게이션

**구조**:
- 상단: 로고, 메뉴 (인라인), 프로필
- 하단 네비게이션 없음

## 5. 이미지 및 미디어

### 반응형 이미지

**Next.js Image 컴포넌트 사용**:
- 자동 최적화
- 적절한 크기로 리사이징
- WebP 포맷 지원

**예시**:
```tsx
<Image
  src="/image.jpg"
  width={800}
  height={600}
  sizes="(max-width: 768px) 100vw, (max-width: 1024px) 50vw, 33vw"
  alt="Description"
/>
```
