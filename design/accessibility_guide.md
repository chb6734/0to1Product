## 접근성 원칙

1. **포용성**: 모든 사용자가 사용할 수 있어야 함
2. **명확성**: 정보와 기능이 명확해야 함
3. **유연성**: 다양한 방법으로 접근 가능해야 함
4. **일관성**: 예측 가능한 인터랙션
5. **오류 방지**: 실수를 방지하고 복구 가능하게

---

## WCAG 2.1 AA 기준 준수

### 레벨 A (필수)

- ✅ 모든 비텍스트 콘텐츠에 대체 텍스트 제공
- ✅ 키보드로 모든 기능 접근 가능
- ✅ 자동 재생 오디오 제어 가능
- ✅ 충분한 색상 대비

### 레벨 AA (목표)

- ✅ 색상 대비 비율 4.5:1 이상 (텍스트)
- ✅ 색상 대비 비율 3:1 이상 (큰 텍스트, UI 컴포넌트)
- ✅ 텍스트 크기 200%까지 확대 가능
- ✅ 키보드 포커스 명확히 표시
- ✅ 에러 메시지 명확히 제공

## 1. 색상 대비 비율

### 텍스트 대비

**일반 텍스트** (18px 미만):
- **대비 비율**: 4.5:1 이상
- **예시**: Gray 900 on White = 15.8:1 ✅

**큰 텍스트** (18px 이상 또는 14px 이상 Bold):
- **대비 비율**: 3:1 이상
- **예시**: Gray 800 on White = 12.6:1 ✅

### UI 컴포넌트 대비

**보더, 그래픽**:
- **대비 비율**: 3:1 이상
- **예시**: Primary 500 on White = 4.5:1 ✅

**상태 표시**:
- **대비 비율**: 3:1 이상
- **예시**: Success 500 on White = 4.5:1 ✅

---

### 다크 모드 대비

**텍스트**:
- **대비 비율**: 4.5:1 이상
- **예시**: Gray 100 on Gray 900 = 15.8:1 ✅

**UI 컴포넌트**:
- **대비 비율**: 3:1 이상
- **예시**: Primary 500 on Gray 900 = 4.5:1 ✅

## 2. 키보드 네비게이션

### 포커스 인디케이터

**스타일**:
- **보더**: Primary 500, 2px 두께
- **그림자**: 0 0 0 3px rgba(99, 102, 241, 0.2)
- **명확성**: 항상 명확하게 표시

**예시**:
```css
.focus-indicator {
  outline: 2px solid #6366F1;
  outline-offset: 2px;
  box-shadow: 0 0 0 3px rgba(99, 102, 241, 0.2);
}
```

---

### 포커스 순서

**논리적 순서**:
- 위에서 아래로
- 왼쪽에서 오른쪽으로
- 주요 콘텐츠 우선

**스킵 링크**:
- 긴 네비게이션 건너뛰기
- 주요 콘텐츠로 바로 이동

**예시**:
```html
<a href="#main-content" class="skip-link">
  주요 콘텐츠로 건너뛰기
</a>
```

---

### 키보드 단축키

**기본 단축키**:
- **Tab**: 다음 요소로 이동
- **Shift + Tab**: 이전 요소로 이동
- **Enter / Space**: 활성화
- **ESC**: 모달 닫기, 메뉴 닫기
- **Arrow Keys**: 메뉴 네비게이션

---

## 3. 스크린 리더 지원

### ARIA 레이블

**버튼**:
- 아이콘만 있는 버튼: aria-label 제공
- 명확한 라벨 사용

**예시**:
```html
<button aria-label="편지 삭제">
  <TrashIcon />
</button>
```

---

### ARIA 속성

**상태 표시**:
- **aria-live**: 동적 콘텐츠 변경 알림
- **aria-atomic**: 전체 메시지 읽기
- **aria-busy**: 로딩 상태 표시

**예시**:
```html
<div aria-live="polite" aria-atomic="true">
  편지가 저장되었습니다.
</div>

<div aria-busy="true">
  저장 중...
</div>
```

---

### 의미론적 HTML

**올바른 HTML 태그 사용**:
- `<button>`: 버튼
- `<a>`: 링크
- `<nav>`: 네비게이션
- `<main>`: 주요 콘텐츠
- `<header>`, `<footer>`: 헤더, 푸터

**예시**:
```html
<nav aria-label="주요 네비게이션">
  <ul>
    <li><a href="/">홈</a></li>
    <li><a href="/inbox">보관함</a></li>
  </ul>
</nav>
```

## 4. 포커스 인디케이터

### 포커스 스타일

**기본 포커스**:
- **보더**: Primary 500, 2px
- **그림자**: 0 0 0 3px rgba(99, 102, 241, 0.2)
- **명확성**: 항상 명확하게 표시

**커스텀 포커스**:
- 브랜드 컬러 사용
- 충분한 대비 확보
- 명확한 시각적 표시

---

### 포커스 트랩

**모달 내부**:
- 모달 열림 시 포커스 트랩
- ESC 키로 닫기
- 외부 클릭으로 닫기

**예시**:
```typescript
// 모달 열림 시 첫 번째 포커스 가능 요소로 포커스 이동
useEffect(() => {
  if (isOpen) {
    firstFocusableElement?.focus()
  }
}, [isOpen])
```

## 5. 색상 의존성 제거

### 정보 전달

**색상만으로 정보 전달 금지**:
- 아이콘 추가
- 텍스트 추가
- 패턴 추가

**예시**:
- ❌ 빨간색만으로 에러 표시
- ✅ 빨간색 + 경고 아이콘 + "에러" 텍스트

---

### 상태 표시

**다양한 방법으로 상태 표시**:
- 색상
- 아이콘
- 텍스트
- 패턴

**예시**:
- 성공: 초록색 + 체크 아이콘 + "성공" 텍스트
- 에러: 빨간색 + X 아이콘 + "에러" 텍스트

## 6. 텍스트 크기 및 확대

### 텍스트 확대

**최소 200%까지 확대 가능**:
- 브라우저 확대 기능 지원
- 반응형 레이아웃 유지
- 텍스트 잘림 방지

**예시**:
```css
html {
  font-size: 16px; /* 기본 크기 */
}

@media (min-width: 768px) {
  html {
    font-size: 18px; /* 데스크톱에서 약간 크게 */
  }
}
```

---

### 가독성 향상

**행간**:
- 본문: 1.5 (24px / 16px)
- 제목: 1.2 - 1.3

**자간**:
- 본문: 기본 (0)
- 제목: -0.02em (약간 좁게)

---

## 7. 애니메이션 제어

### Reduced Motion

**미디어 쿼리 지원**:
- 사용자 설정 존중
- 애니메이션 비활성화 또는 단순화

**예시**:
```css
@media (prefers-reduced-motion: reduce) {
  * {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
  }
}
```

---

### 자동 재생 방지

**비디오, 오디오**:
- 자동 재생 비활성화
- 사용자 제어 제공
- 일시정지/재생 버튼 제공

---

## 8. 폼 접근성

### 라벨 연결

**명확한 라벨**:
- `<label>` 태그 사용
- `for` 속성으로 연결
- 또는 `aria-label` 사용

**예시**:
```html
<label for="nickname">닉네임</label>
<input type="text" id="nickname" name="nickname" />

<!-- 또는 -->
<input type="text" aria-label="닉네임" />
```

---

### 에러 메시지

**명확한 에러 메시지**:
- 에러 발생 시 즉시 알림
- 에러 위치 명시
- 해결 방법 제시

**예시**:
```html
<input type="email" aria-invalid="true" aria-describedby="email-error" />
<span id="email-error" role="alert">
  올바른 이메일 주소를 입력해주세요.
</span>
```

---

### 필수 필드 표시

**명확한 표시**:
- 별표(*) 사용
- "필수" 텍스트 추가
- `aria-required="true"` 속성

**예시**:
```html
<label for="nickname">
  닉네임 <span aria-label="필수">*</span>
</label>
<input type="text" id="nickname" aria-required="true" />
```
