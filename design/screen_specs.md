## 화면 명세 구조

각 화면에 대해 다음을 명세합니다:
1. 레이아웃 구조
2. 콘텐츠 계층 구조

## 화면 1: 랜딩 페이지 (/)

### 레이아웃 구조

```
┌─────────────────────────────────────┐
│  [로고] [보관함] [둘러보기] [로그인] │
│  상단 네비게이션 (sticky)            │
├─────────────────────────────────────┤
│                                     │
│         히어로 섹션                  │
│    "어떤 플랫폼이든                  │
│     음악을 공유하세요"               │
│    [편지 만들기] [데모 보기]        │
│                                     │
├─────────────────────────────────────┤
│      추천 편지 섹션                  │
│  [편지 카드 1] [편지 카드 2] [편지 카드 3]│
├─────────────────────────────────────┤
│      이용 방법 섹션                  │
│  [1단계] [2단계] [3단계]            │
├─────────────────────────────────────┤
│      CTA 섹션                       │
│    "오늘부터 음악을 공유하세요"      │
│    [첫 편지 만들기]                 │
└─────────────────────────────────────┘
```

### 콘텐츠 계층 구조

**Level 1 (최우선)**:
- 히어로 메시지: H1 (48px-60px), Bold, White
- CTA 버튼: Primary Button (노란색), 큰 크기

**Level 2 (차순위)**:
- 추천 편지: H2 (30px), 카드 그리드 (3 컬럼)
- 이용 방법: H2 (30px), 3단계 카드
- CTA 섹션: H2 (30px), 그라데이션 배경

**Level 3 (보조)**:
- 네비게이션: 로고 (노란색 아이콘), 메뉴 (보관함, 둘러보기)
- 배지: 플랫폼 중립성 배지

### 인터랙션 요소 배치

- **상단 네비게이션**: sticky, 반투명 배경 (`rgba(18, 18, 18, 0.8)`)
- **편지 만들기 버튼**: 히어로 섹션, 노란색 배경
- **편지 카드**: 호버 시 opacity 변경
- **스크롤**: 부드러운 스크롤

### 상태별 디자인

**Default**:
- 배경: `#0A0A0A` (다크)
- 버튼: `#FFE11D` (노란색), 텍스트 Black
- 카드: `#121212` 배경, `rgba(255, 255, 255, 0.05)` 보더

**Hover**:
- 버튼: Opacity 0.9
- 카드: Opacity 0.9
- 링크: 텍스트 White로 변경

**Loading**:
- 스켈레톤 UI: 다크 배경에 회색 플레이스홀더

### 반응형 레이아웃

**Mobile (< 768px)**:
- 네비게이션: 메뉴 숨김 (햄버거 메뉴)
- 히어로: 세로 레이아웃, 버튼 전체 너비
- 카드: 1 컬럼

**Desktop (> 1024px)**:
- 네비게이션: 전체 메뉴 표시
- 히어로: 가로 레이아웃, 버튼 인라인
- 카드: 3 컬럼

## 화면 2: 로그인 페이지 (/login)

### 레이아웃 구조

```
┌─────────────────────────────────────┐
│                                     │
│    [노란색 배경 로고 아이콘]         │
│                                     │
│  "FAN:STAGE에 오신 것을 환영합니다" │
│      (H1, 48px, White)              │
│                                     │
│  "소셜 계정으로 간편하게 시작하세요" │
│      (Body, 16px, Gray)             │
│                                     │
│    ┌─────────────────────────┐    │
│    │  [Google 아이콘] Google  │    │
│    └─────────────────────────┘    │
│                                     │
│    ┌─────────────────────────┐    │
│    │  [Kakao 아이콘] Kakao    │    │
│    └─────────────────────────┘    │
│                                     │
│    ┌─────────────────────────┐    │
│    │  [Apple 아이콘] Apple    │    │
│    └─────────────────────────┘    │
│                                     │
│  "계속 진행하면 이용약관에 동의..."  │
│  [이용약관] [개인정보처리방침]       │
│                                     │
└─────────────────────────────────────┘
```

### 콘텐츠 계층 구조

**Level 1**:
- 로고: 중앙 상단, 노란색 배경 (`#FFE11D`), 음악 아이콘
- 헤더: H1 (48px), White, 중앙

**Level 2**:
- 소셜 로그인 버튼: 전체 너비, 높이 48px, 각 플랫폼 브랜드 컬러

**Level 3**:
- 안내 텍스트: Body Small (14px), Gray, 중앙
- 약관 링크: Body Small, Secondary 컬러

### 인터랙션 요소 배치

- **소셜 로그인 버튼**: 세로 배치, 간격 12px
- **각 버튼**: 플랫폼 브랜드 컬러 사용 (Google: 흰색/파란색, Kakao: 노란색, Apple: 검은색)
- **로딩**: 버튼 클릭 시 console.log (퍼블리싱 테스트용)

### 상태별 디자인

**Default**:
- 배경: `#0A0A0A` (다크)
- 버튼: 각 플랫폼 브랜드 컬러
- 텍스트: White (버튼 내부)

**Hover**:
- 버튼: Opacity 0.9
- 커서: pointer

**Error**:
- 토스트 에러: 상단에 표시 (현재 비활성화)

### 반응형 레이아웃

**Mobile**:
- 버튼: 전체 너비
- 패딩: 16px (px-4)

**Desktop**:
- 버튼: 최대 너비 400px, 중앙 정렬
- 패딩: 32px (py-32)

## 화면 3: 온보딩 페이지 (/onboarding)

### 레이아웃 구조

```
┌─────────────────────────────────────┐
│                                     │
│    "프로필 설정" (H1, 48px, White)   │
│                                     │
│  "닉네임과 프로필 사진을 설정해주세요"│
│      (Body, 16px, Gray)             │
│                                     │
│    ┌─────────────────────────┐    │
│    │                         │    │
│    │  [그라데이션 원형]        │    │
│    │  "?" (중앙)              │    │
│    │  [노란색 플러스 버튼]     │    │
│    │                         │    │
│    └─────────────────────────┘    │
│    "프로필 사진 (선택)"             │
│                                     │
│    닉네임 *                         │
│    ┌─────────────────────────┐    │
│    │ 닉네임을 입력하세요      │    │
│    └─────────────────────────┘    │
│    0/20                             │
│                                     │
│    [시작하기] (비활성화: opacity 0.5)│
│                                     │
└─────────────────────────────────────┘
```

### 콘텐츠 계층 구조

**Level 1**:
- 헤더: H1 (48px), White, 중앙
- 서브 텍스트: Body (16px), Gray, 중앙

**Level 2**:
- 프로필 이미지: 그라데이션 원형 배경 (`#FFE11D` → `#2ADFFF`), 96x96px
- 닉네임 입력: 다크 배경 (`#1A1A1A`), 보더 `rgba(255, 255, 255, 0.1)`

**Level 3**:
- 시작하기 버튼: 노란색 (`#FFE11D`), 비활성화 시 opacity 0.5

### 인터랙션 요소 배치

- **프로필 이미지**: 그라데이션 원형 배경, 중앙에 "?" 텍스트, 우하단 노란색 플러스 버튼
- **닉네임 입력**: 자동 포커스, 실시간 문자 수 카운터 (0/20)
- **시작하기 버튼**: 닉네임 없으면 비활성화

### 상태별 디자인

**Default**:
- 배경: `#0A0A0A` (다크)
- 카드: `#121212` 배경, `rgba(255, 255, 255, 0.05)` 보더
- 입력 필드: `#1A1A1A` 배경, `rgba(255, 255, 255, 0.1)` 보더

**Focus**:
- 입력 필드: 포커스 링 표시

**Disabled**:
- 시작하기 버튼: Opacity 0.5, cursor not-allowed

### 반응형 레이아웃

**Mobile**:
- 입력 필드: 전체 너비
- 프로필 이미지: 중앙 정렬
- 버튼: 전체 너비

**Desktop**:
- 최대 너비: 400px, 중앙 정렬
- 프로필 이미지: 중앙 정렬
- 버튼: 전체 너비

### 레이아웃 구조 (데스크톱)

```

│  [로고] [보관함] [둘러보기] [편지 만들기] [프로필] │

│                          │                         │

│                          │                         │
│  "음악 편지 만들기"       │  ┌──────────────────┐ │
│  "곡을 추가하고..."       │  │  미리보기         │ │
```

### 단계 2: 완성 화면

```
┌─────────────────────────────────────┐
│  [로고] [보관함] [둘러보기] [편지 만들기] [프로필] │

│                                     │
│  "편지가 완성되었습니다!"            │
│  "링크나 QR코드로 공유하세요"        │
│                                     │
│  ┌─────────────────────────┐       │
│  │ 공유 링크               │       │
│  │ [링크 입력] [복사]      │       │
│  └─────────────────────────┘       │
│                                     │
│  ┌─────────────────────────┐       │
│  │ QR 코드                 │       │

│  └─────────────────────────┘       │
│                                     │
│  ┌─────────────────────────┐       │
│  │ 전체 공개               │       │
│  └─────────────────────────┘       │
│                                     │
│  [수정하기] [완료]                  │
│                                     │
└─────────────────────────────────────┘
```

### 콘텐츠 계층 구조

**단계 1 - Level 1**:
- 헤더: H2 (32px), White
- 서브 텍스트: Body (16px), Gray

**단계 1 - Level 2**:
- 곡 추가 카드: 다크 배경 (`#121212`), 검색 입력 필드
- 메시지 카드: 다크 배경, 텍스트 영역, 문자 수 카운터, 템플릿 사용 버튼
- 공개 설정 카드: 다크 배경, 체크박스

**단계 1 - Level 3**:
- 다음 버튼: 노란색 (`#FFE11D`), 메시지 없으면 비활성화 (opacity 0.5)
- 미리보기: 빈 상태 메시지

**단계 2 - Level 1**:
- 헤더: H2 (32px), White
- 서브 텍스트: Body (16px), Gray

**단계 2 - Level 2**:
- 공유 링크 카드: 다크 배경, 링크 입력 필드, 복사 버튼 (노란색)
- QR 코드 카드: 다크 배경, QR 코드 이미지, 설명 텍스트
- 전체 공개 카드: 다크 배경, 아이콘, 설명

**단계 2 - Level 3**:
- 수정하기 버튼: 아웃라인 스타일
- 완료 버튼: 초록색 (`#10B981`)

### 인터랙션 요소 배치

**단계 1**:
- 검색 입력: 검색 아이콘 포함, 다크 배경 (`#1A1A1A`)
- 메시지 입력: 실시간 문자 수 카운터 (0/500)
- 템플릿 사용: 시안색 (`#2ADFFF`) 링크
- 다음 버튼: 메시지 없으면 비활성화

**단계 2**:
- 링크 복사: 노란색 버튼, 클립보드 복사
- 수정하기: 첫 번째 단계로 돌아가기
- 완료: 완료 처리

### 상태별 디자인

**Default**:
- 배경: `#0A0A0A` (다크)
- 카드: `#121212` 배경, `rgba(255, 255, 255, 0.05)` 보더
- 입력 필드: `#1A1A1A` 배경, `rgba(255, 255, 255, 0.1)` 보더

**Hover**:
- 버튼: Opacity 0.9
- 카드: Opacity 0.9

**Disabled**:
- 다음 버튼: Opacity 0.5, cursor not-allowed

### 반응형 레이아웃

**Mobile**:
- 단일 컬럼 레이아웃
- 카드: 전체 너비
- 버튼: 전체 너비

**Desktop**:
- 단계 1: 분할 레이아웃 (50% 편집 / 50% 미리보기)
- 단계 2: 단일 컬럼, 최대 너비 768px

## 화면 5: 편지 열람 페이지 (/letter/[token])

### 레이아웃 구조

```
┌─────────────────────────────────────┐
│                                     │
│         배경 (그라데이션)            │
│                                     │
│    ┌─────────────────────────┐   │
│    │                         │   │
│    │      편지 카드           │   │
│    │                         │   │
│    │  [프로필] 보낸 사람      │   │
│    │                         │   │
│    │  [메시지]                │   │
│    │                         │   │
│    │  [플레이리스트]           │   │
│    │  ┌───────────────────┐ │   │
│    │  │ 곡 1 [앨범 커버]   │ │   │
│    │  │ 곡 2 [앨범 커버]   │ │   │
│    │  └───────────────────┘ │   │
│    │                         │   │
│    │  [내 플랫폼에 저장]      │   │
│    │                         │   │
│    └─────────────────────────┘   │
│                                     │
└─────────────────────────────────────┘
```

### 콘텐츠 계층 구조

**Level 1**:
- 편지 카드: 중앙 정렬, 큰 크기
- 배경: 그라데이션 또는 패턴

**Level 2**:
- 보낸 사람 정보: 프로필 이미지, 닉네임
- 메시지: 큰 텍스트, 감성적 디자인

**Level 3**:
- 플레이리스트: 곡 목록, 앨범 커버
- CTA 버튼: 큰 크기, Primary 컬러

### 인터랙션 요소 배치

- **편지 카드**: 중앙 정렬, 그림자
- **플레이리스트**: 그리드 레이아웃
- **CTA 버튼**: 하단 고정 (모바일), 카드 내부 (데스크톱)
- **스크롤**: 부드러운 스크롤

### 상태별 디자인

**Default**:
- 편지 카드: White 배경, 그림자
- 배경: 그라데이션 (Primary 50 → Secondary 50)

**Hover**:
- 곡 카드: 확대 효과 (Scale 1.05)
- CTA 버튼: Primary 600

**Loading**:
- 편지 로드: 스켈레톤 UI
- 저장 중: 진행 표시기

**Success**:
- 저장 완료: 성공 애니메이션 (체크마크)
- 플랫폼 링크: Secondary Button

**Error**:
- 편지 없음: 404 페이지
- 저장 실패: 토스트 에러, 재시도 버튼

### 반응형 레이아웃

**Mobile**:
- 편지 카드: 전체 너비, 패딩 24px
- 플레이리스트: 1 컬럼

### 레이아웃 구조

```
┌─────────────────────────────────────┐
│  [로고] [보관함] [둘러보기] [편지 만들기] [프로필] │
├─────────────────────────────────────┤
│  [← 뒤로 가기]                       │
├─────────────────────────────────────┤
│                                     │
│  ┌─────────────────────────┐       │
│  │ [큰 프로필] 보낸 사람   │       │
│  │ 날짜                    │       │
│  │                         │       │
│  │ [메시지 영역]           │       │
│  │ "요즘 날씨가 추워지면서..."│       │
│  │                         │       │
│  │ [통계] 15곡 1245회 재생 89개│       │
│  └─────────────────────────┘       │
│                                     │
│  ┌─────────────────────────┐       │
│  │ 플레이리스트            │       │

│  │                         │       │
│  │ 1. [앨범] Winter Song   │       │
│  │    The Dreamers         │       │
│  │ 2. [앨범] Snowflake     │       │
│  │    Coldplay             │       │
│  └─────────────────────────┘       │
│                                     │
└─────────────────────────────────────┘
```

### 콘텐츠 계층 구조

**Level 1**:
- 뒤로 가기 버튼: Gray 텍스트
- 편지 카드: 다크 배경 (`#121212`)

**Level 2**:
- 보낸 사람 정보: 큰 프로필 이미지 (64x64px), 이름 (H2, 32px), 날짜
- 메시지 영역: 다크 배경 (`#1A1A1A`), 보더
- 통계: 곡 수, 재생 횟수, 좋아요 수

**Level 3**:
- 플레이리스트 섹션: 헤딩, 전체 재생 버튼 (노란색)
- 곡 목록: 번호, 앨범 커버 (노란색 배경), 제목, 아티스트, 재생 버튼

### 인터랙션 요소 배치

- **뒤로 가기**: 이전 페이지로 이동
- **전체 재생**: 노란색 버튼
- **곡 재생**: 각 곡의 재생 버튼

### 상태별 디자인

**Default**:
- 배경: `#0A0A0A` (다크)
- 편지 카드: `#121212` 배경, `rgba(255, 255, 255, 0.05)` 보더
- 메시지 영역: `#1A1A1A` 배경, `rgba(255, 255, 255, 0.1)` 보더

**Hover**:
- 버튼: Opacity 0.9
- 곡 항목: Opacity 0.9

### 반응형 레이아웃

**Mobile**:
- 편지 카드: 전체 너비
- 플레이리스트: 1 컬럼

**Desktop**:
- 편지 카드: 최대 너비 1024px, 중앙 정렬
- 플레이리스트: 전체 너비
