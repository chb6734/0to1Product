## 화면 명세 구조

각 화면에 대해 다음을 명세합니다:
1. 레이아웃 구조
2. 콘텐츠 계층 구조
3. 인터랙션 요소 배치

5. 반응형 레이아웃

## 화면 1: 랜딩 페이지 (/)

### 레이아웃 구조

```
┌─────────────────────────────────────┐
│         상단 네비게이션              │
│  [로고]              [로그인] [시작하기]│
├─────────────────────────────────────┤
│                                     │
│         히어로 섹션                  │
│    "음악으로 모인 사람들이..."       │
│         [시작하기 버튼]              │
│                                     │
├─────────────────────────────────────┤
│      핵심 가치 제안 섹션             │
│  [플랫폼 중립성] [감성 전달] [영구보관]│
├─────────────────────────────────────┤
│      미리보기 섹션                   │
│      [편지 카드 예시]                │
├─────────────────────────────────────┤
│            푸터                      │
└─────────────────────────────────────┘
```

### 콘텐츠 계층 구조

**Level 1 (최우선)**:
- 히어로 메시지: H1 (36px), Bold
- CTA 버튼: Primary Button, 큰 크기

**Level 2 (차순위)**:
- 핵심 가치 제안: H2 (30px), 카드 형태
- 미리보기: 편지 카드 예시

**Level 3 (보조)**:
- 네비게이션: 로고, 메뉴
- 푸터: 링크, 저작권

### 인터랙션 요소 배치

- **상단 네비게이션**: 고정, 그림자
- **시작하기 버튼**: 히어로 섹션 중앙, 큰 크기
- **스크롤**: 부드러운 스크롤, 섹션별 애니메이션

### 상태별 디자인

**Default**:
- 배경: 그라데이션 (Primary 50 → Secondary 50)
- 버튼: Primary 500

**Hover**:
- 버튼: Primary 600, Scale (1.05)
- 카드: 그림자 증가

**Loading**:
- 스켈레톤 UI: 히어로 섹션, 카드

**Error**:
- 에러 메시지: 토스트 형태

### 반응형 레이아웃

**Mobile (< 768px)**:
- 네비게이션: 햄버거 메뉴
- 히어로: 세로 레이아웃, 버튼 전체 너비
- 카드: 1 컬럼

**Desktop (> 1024px)**:
- 네비게이션: 전체 메뉴 표시
- 히어로: 가로 레이아웃, 버튼 인라인
- 카드: 3 컬럼

```

│    │  [Apple 아이콘] Apple    │    │

└─────────────────────────────────────┘
```

### 콘텐츠 계층 구조

**Level 1**:
- 로고: 중앙 상단
- 헤더: H2 (30px), 중앙

**Level 2**:
- 소셜 로그인 버튼: 전체 너비, 높이 56px

**Level 3**:
- 안내 텍스트: Body Small (14px), 중앙

### 인터랙션 요소 배치

- **소셜 로그인 버튼**: 세로 배치, 간격 12px
- **각 버튼**: 플랫폼 브랜드 컬러 사용
- **로딩**: 버튼 클릭 시 스피너 표시

### 상태별 디자인

**Default**:
- 버튼: 각 플랫폼 브랜드 컬러
- 텍스트: White (버튼 내부)

**Hover**:
- 버튼: 밝기 증가 (brightness 1.1)
- 커서: pointer

**Active**:
- 버튼: Scale (0.98)
- 스피너: 버튼 내부 표시

**Loading**:
- 버튼: 비활성화, 스피너 표시
- 다른 버튼: 비활성화

**Error**:
- 토스트 에러: 상단에 표시
- 재시도 버튼: 에러 메시지 내부

### 반응형 레이아웃

**Mobile**:
- 버튼: 전체 너비
- 패딩: 24px

**Desktop**:
- 버튼: 최대 너비 400px, 중앙 정렬
- 패딩: 32px

## 화면 3: 온보딩 페이지 (/onboarding)

### 레이아웃 구조

```
┌─────────────────────────────────────┐
│  [←]  진행 표시기 [Step 1/2]        │
├─────────────────────────────────────┤
│                                     │
│    "프로필을 설정해주세요" (H2)      │
│                                     │
│    닉네임                            │
│    ┌─────────────────────────┐    │
│    │                         │    │
│    └─────────────────────────┘    │
│                                     │
│    프로필 이미지 (선택)              │
│    ┌─────────────────────────┐    │
│    │    [이미지 업로드 영역]   │    │
│    │    "클릭하거나 드래그"    │    │
│    └─────────────────────────┘    │
│                                     │
│    [나중에]              [완료]     │
│                                     │
└─────────────────────────────────────┘
```

### 콘텐츠 계층 구조

**Level 1**:
- 진행 표시기: 상단, Primary 컬러
- 헤더: H2 (30px)

**Level 2**:
- 입력 필드: 큰 크기 (높이 56px)
- 프로필 이미지: 원형, 큰 크기 (128px)

**Level 3**:
- 버튼: 하단 고정, Ghost / Primary

### 인터랙션 요소 배치

- **진행 표시기**: 상단 고정, Primary 컬러
- **닉네임 입력**: 자동 포커스, 실시간 검증
- **프로필 이미지**: 드래그 앤 드롭, 클릭 업로드
- **버튼**: 하단 고정 (모바일), 오른쪽 정렬 (데스크톱)

### 상태별 디자인

**Default**:
- 입력 필드: Gray 300 보더
- 프로필 이미지: Gray 200 배경, 점선 보더

**Focus**:
- 입력 필드: Primary 500 보더, 그림자
- 프로필 이미지: Primary 100 배경

**Error**:
- 입력 필드: Error 500 보더
- 에러 메시지: 인라인, Error 500 텍스트

**Success**:
- 입력 필드: Success 500 보더 (검증 통과)
- 체크 아이콘: 입력 필드 오른쪽

**Loading**:
- 업로드 중: 스피너 표시
- 완료 버튼: 로딩 스피너

### 반응형 레이아웃

**Mobile**:
- 입력 필드: 전체 너비
- 프로필 이미지: 중앙 정렬
- 버튼: 전체 너비, 세로 배치

**Desktop**:
- 입력 필드: 최대 너비 400px
- 프로필 이미지: 중앙 정렬
- 버튼: 인라인, 오른쪽 정렬

### 레이아웃 구조 (모바일)

```
┌─────────────────────────────────────┐
│  [←]  진행 표시기 [Step 1/3]        │
├─────────────────────────────────────┤
│  [편집] [미리보기] (탭)              │
├─────────────────────────────────────┤
│                                     │
│  ┌─────────────────────────┐       │
│  │  [검색 바]              │       │
│  └─────────────────────────┘       │
│                                     │
│  [플레이리스트]                      │
│  ┌─────────────────────────┐       │
│  │  곡 1 [삭제]            │       │
│  │  곡 2 [삭제]            │       │

│  ┌─────────────────────────┐       │
│  │                         │       │
│  └─────────────────────────┘       │
│  [AI 생성] [다음]                   │
│                                     │
└─────────────────────────────────────┘
```

### 콘텐츠 계층 구조

**Level 1**:
- 진행 표시기: 상단 고정
- 검색 바: 상단 고정

**Level 2**:
- 플레이리스트: 곡 목록, 드래그 앤 드롭
- 메시지 입력: 큰 텍스트 영역

**Level 3**:
- 미리보기: 우측 (데스크톱), 탭 (모바일)
- 버튼: 하단 고정

### 인터랙션 요소 배치

- **검색 바**: 상단 고정, debounce 300ms
- **곡 추가**: 클릭 시 애니메이션 (Scale + Fade)
- **드래그 앤 드롭**: react-beautiful-dnd
- **자동 저장**: 하단에 "자동 저장됨" 표시
- **미리보기**: 실시간 업데이트

### 상태별 디자인

**Default**:
- 검색 바: Gray 300 보더
- 곡 카드: White 배경, Gray 200 보더
- 메시지 입력: Gray 300 보더

**Hover**:
- 곡 카드: 그림자 증가, 삭제 버튼 표시
- 검색 결과: 호버 시 배경 Gray 50

**Active**:
- 드래그 중: 투명도 감소, 그림자 증가
- 입력 필드: Primary 500 보더

**Loading**:
- 검색: 스켈레톤 UI
- 저장: 하단에 스피너
- AI 생성: 버튼 내부 스피너

**Error**:
- 검색 실패: "다시 시도해주세요" 메시지
- 저장 실패: 토스트 에러
- 유효성 검사: 인라인 에러 메시지

### 반응형 레이아웃

**Mobile**:
- 탭으로 전환: 편집 / 미리보기
- 검색 바: 전체 너비
- 곡 카드: 전체 너비
- 버튼: 하단 고정, 전체 너비

**Desktop**:
- 분할 레이아웃: 편집 (60%) / 미리보기 (40%)
- 검색 바: 편집 영역 내부
- 곡 카드: 그리드 레이아웃 (2 컬럼)
- 버튼: 하단, 오른쪽 정렬

│    │      편지 카드           │   │

│    │                         │   │

│    │                         │   │

### 콘텐츠 계층 구조

- 보낸 사람 정보: 프로필 이미지, 닉네임

- 플레이리스트: 곡 목록, 앨범 커버

- **편지 카드**: 중앙 정렬, 그림자
- **플레이리스트**: 그리드 레이아웃
- **CTA 버튼**: 하단 고정 (모바일), 카드 내부 (데스크톱)
- **스크롤**: 부드러운 스크롤

### 상태별 디자인

**Default**:
- 편지 카드: White 배경, 그림자
- 배경: 그라데이션 (Primary 50 → Secondary 50)

**Hover**:
- 곡 카드: 확대 효과 (Scale 1.05)
- CTA 버튼: Primary 600

**Loading**:
- 편지 로드: 스켈레톤 UI
- 저장 중: 진행 표시기

**Success**:
- 저장 완료: 성공 애니메이션 (체크마크)
- 플랫폼 링크: Secondary Button

**Error**:
- 편지 없음: 404 페이지
- 저장 실패: 토스트 에러, 재시도 버튼

### 반응형 레이아웃

**Mobile**:
- 편지 카드: 전체 너비, 패딩 24px
- 플레이리스트: 1 컬럼
- CTA 버튼: 하단 고정, 전체 너비

**Desktop**:
- 편지 카드: 최대 너비 600px, 중앙 정렬
- 플레이리스트: 2 컬럼
- CTA 버튼: 카드 내부, 인라인

## 화면 6: 보관함 페이지 (/inbox, /sent)

### 레이아웃 구조

```
┌─────────────────────────────────────┐
│  상단 네비게이션                     │
├─────────────────────────────────────┤
│  [받은 편지] [보낸 편지] (탭)        │
├─────────────────────────────────────┤
│  [검색 바]  [필터] [그리드/리스트]   │
├─────────────────────────────────────┤
│                                     │
│  ┌──────┐  ┌──────┐  ┌──────┐    │
│  │ 편지1 │  │ 편지2 │  │ 편지3 │    │
│  └──────┘  └──────┘  └──────┘    │
│                                     │
│  ┌──────┐  ┌──────┐  ┌──────┐    │
│  │ 편지4 │  │ 편지5 │  │ 편지6 │    │
│  └──────┘  └──────┘  └──────┘    │
│                                     │
│  [더 보기] (무한 스크롤)            │
│                                     │
└─────────────────────────────────────┘
```

### 콘텐츠 계층 구조

**Level 1**:
- 탭: 상단 고정, 활성 탭 강조
- 검색 바: 상단 고정

**Level 2**:
- 필터: 칩 형태, 선택 시 Primary 컬러
- 뷰 전환: 그리드/리스트 토글

**Level 3**:
- 편지 목록: 그리드 또는 리스트 뷰
- 무한 스크롤: 하단 스크롤 시 로드

### 인터랙션 요소 배치

- **탭**: 상단 고정, 클릭 시 즉시 전환
- **검색**: debounce 300ms, 실시간 결과
- **필터**: 클릭 시 즉시 적용
- **뷰 전환**: 애니메이션 효과
- **편지 카드**: 클릭 시 상세 페이지

### 상태별 디자인

**Default**:
- 탭: Gray 200 배경 (비활성), White 배경 (활성)
- 편지 카드: White 배경, 그림자

**Hover**:
- 편지 카드: 그림자 증가, Scale (1.02)
- 탭: Gray 100 배경

**Active**:
- 탭: Primary 500 배경, White 텍스트
- 필터: Primary 500 배경, White 텍스트

**Loading**:
- 편지 목록: 스켈레톤 UI
- 무한 스크롤: 하단에 스피너

**Empty**:
- 빈 상태: 중앙 정렬, 아이콘 + 메시지
- "첫 편지를 만들어보세요" 안내

### 반응형 레이아웃

**Mobile**:
- 탭: 전체 너비, 세로 배치
- 검색 바: 전체 너비
- 편지 카드: 1 컬럼 (그리드), 전체 너비 (리스트)

**Desktop**:
- 탭: 인라인, 상단 고정
- 검색 바: 최대 너비 400px
- 편지 카드: 3 컬럼 (그리드), 2 컬럼 (리스트)

## 공통 컴포넌트

### 상단 네비게이션

**구조**:
- 로고: 왼쪽
- 메뉴: 오른쪽 (데스크톱), 햄버거 (모바일)
- 프로필: 오른쪽 끝

**상태**:
- 고정: 스크롤 시 그림자 추가
- 모바일: 햄버거 메뉴, 드로어

### 하단 네비게이션 (모바일)

**구조**:
- 홈, 보관함, 생성, 프로필
- 각 항목: 아이콘 + 라벨

**상태**:
- 활성: Primary 500 컬러
- 비활성: Gray 500 컬러

### Floating Action Button (FAB)

**위치**: 우측 하단 고정

**상태**:
- Default: Primary 500 배경, White 아이콘
- Hover: Primary 600 배경, Scale (1.1)
- Active: Primary 700 배경, Scale (0.95)
