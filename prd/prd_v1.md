## 1. 제품 개요

### 비전과 목표

**비전**: "음악으로 모인 사람들이, 음악으로 살아갈 수 있도록"

**MVP 목표**: 플랫폼 중립적 음악 편지 서비스를 통해 핵심 가치를 검증한다.

### 핵심 가치 제안

1. **플랫폼 중립성**: 어떤 음악 스트리밍 서비스를 사용하든 음악을 공유할 수 있다
2. **감성 전달**: 단순 링크가 아닌 메시지와 함께 음악을 전달한다
3. **영구 보관**: 받은 편지를 영구적으로 보관하고 다시 찾을 수 있다

### 타겟 사용자

#### 1차 타겟: 일반 팬 & 음악 애호가 (20-35세)
- **페르소나**: 음악을 좋아하고 친구/연인에게 추천하고 싶은 사람
- **니즈**: 
  - 플랫폼이 달라도 음악을 공유하고 싶음
  - 의미 있는 방식으로 음악을 선물하고 싶음
- **사용 시나리오**: 생일 선물, 위로, 응원, 기념일

#### 2차 타겟: 아티스트 & 음악인
- **페르소나**: 인디/신인 아티스트, 세션 뮤지션
- **니즈**:
  - 팬과 소통하고 싶음
  - 셋리스트를 효과적으로 공유하고 싶음
- **사용 시나리오**: 페스티벌 셋리스트 공유, 신곡 소개

## 2. 기능 명세

### 우선순위 정의
- **P0 (필수)**: MVP의 핵심 가치를 검증하기 위해 반드시 필요
- **P1 (중요)**: 사용자 경험을 크게 향상시키지만 MVP에서는 선택적
- **P2 (향후)**: Phase 2에서 추가

### P0: 핵심 기능

#### 2.1 사용자 인증 및 프로필

**기능 설명**:
- 소셜 로그인을 통한 간편 가입
- 기본 프로필 설정

**상세 요구사항**:
- 소셜 로그인 지원: Google, Kakao, Apple
- 닉네임 설정 (필수)
- 프로필 이미지 업로드 (선택)
- 이메일 주소 (소셜 로그인에서 자동 수집)

**사용자 시나리오**:
1. 사용자가 "시작하기" 버튼 클릭
2. 소셜 로그인 선택 (Google/Kakao/Apple)
3. 권한 승인 후 자동 가입 완료
4. 닉네임 입력 화면으로 이동
5. 프로필 이미지 선택 (선택적)
6. 메인 화면으로 이동

**성공 기준**:
- 가입 완료까지 30초 이내
- 가입 전환율 70% 이상

---

#### 2.2 음악 편지 생성

**기능 설명**:
- 플레이리스트를 만들고 메시지를 추가하여 음악 편지를 생성

**상세 요구사항**:

**2.2.1 곡 검색 및 추가**
- 곡 검색 (제목, 아티스트명)
- 검색 결과 표시 (제목, 아티스트, 앨범 커버)
- 플레이리스트에 곡 추가
- 곡 삭제
- 곡 순서 변경 (드래그 앤 드롭)

**2.2.2 메시지 작성**
- 메시지 입력 (텍스트, 최대 500자)
- AI 메시지 생성 (선택적)
  - 템플릿 기반 (생일, 위로, 축하, 사랑, 우정)
  - 사용자가 수정 가능
- 각 곡마다 메모 추가 (선택적, 최대 100자)

**2.2.3 편지 미리보기**
- 카드 형태로 미리보기
- 모바일/데스크톱 반응형

**사용자 시나리오**:
1. "편지 만들기" 버튼 클릭
2. 곡 검색 및 추가 (최소 1곡)
3. 메시지 작성 또는 AI 생성 선택
4. 각 곡에 메모 추가 (선택적)
5. 미리보기 확인
6. "편지 완성" 클릭

**성공 기준**:
- 편지 생성 완료까지 3분 이내
- 사용자 만족도 4.0/5.0 이상

---

#### 2.3 플랫폼 연동

**기능 설명**:
- 여러 음악 스트리밍 플랫폼과 연동하여 사용자의 플랫폼에 플레이리스트 저장

**상세 요구사항**:

**2.3.1 지원 플랫폼**
- Spotify (OAuth 2.0)
- YouTube Music (OAuth 2.0)
- Apple Music (MusicKit JS)
- 멜론 (사용자 수동 입력 - API 부재)

**2.3.2 곡 매칭 알고리즘**
- 곡 제목 + 아티스트명으로 매칭
- 매칭 실패 시 사용자에게 알림
- 대안 곡 제시 (선택적)

**2.3.3 플레이리스트 저장**
- 사용자가 선택한 플랫폼에 플레이리스트 생성
- 원클릭 저장
- 저장 성공/실패 피드백

**사용자 시나리오**:
1. 편지를 받은 사용자가 링크 클릭
2. 편지 열람
3. "내 플랫폼에 저장" 버튼 클릭
4. 플랫폼 선택 (Spotify/YouTube/Apple Music)
5. 로그인 (처음 사용 시)
6. 플레이리스트 저장 완료

**성공 기준**:
- 플레이리스트 저장 성공률 60% 이상
- 저장 완료까지 10초 이내

---

#### 2.4 편지 전송 및 수신

**기능 설명**:
- 편지를 생성한 후 수신자에게 전송하고, 받은 편지를 관리

**상세 요구사항**:

**2.4.1 편지 전송**
- 수신자 이메일 입력
- 또는 링크 공유 (카카오톡, 문자, SNS)
- 전송 완료 알림

**2.4.2 편지 열람**
- 링크를 통한 편지 열람 (로그인 불필요)
- 카드 형태로 표시
- 모바일 최적화

**2.4.3 편지 보관함**
- 받은 편지 보관함
- 보낸 편지 보관함
- 날짜별 정렬
- 검색 기능 (보낸 사람, 곡 제목)

**사용자 시나리오**:
1. 편지 생성 완료 후 "전송하기" 클릭
2. 수신자 이메일 입력 또는 링크 복사
3. 전송 완료
4. 수신자가 링크 클릭하여 편지 열람
5. 받은 편지는 자동으로 보관함에 저장 (로그인 시)

**성공 기준**:
- 편지 열람률 70% 이상
- 편지 재방문률 30% 이상

## 3. 기술 요구사항

### 기술 스택

#### Frontend
- **Framework**: Next.js 14 (App Router)
- **Language**: TypeScript
- **Styling**: Tailwind CSS
- **UI Components**: shadcn/ui 또는 Tailwind UI
- **State Management**: React Context 또는 Zustand
- **Form Handling**: React Hook Form

#### Backend
- **Database**: Supabase (PostgreSQL)
- **Authentication**: Supabase Auth
- **Storage**: Supabase Storage (프로필 이미지)
- **API**: Next.js API Routes

#### External APIs
- **Spotify**: Spotify Web API
- **YouTube Music**: YouTube Data API v3
- **Apple Music**: MusicKit JS
- **AI**: OpenAI API (GPT-4o Mini) - 선택적

#### 배포 및 인프라
- **Hosting**: Vercel
- **Database**: Supabase (무료 플랜)
- **CDN**: Vercel Edge Network
- **Monitoring**: Vercel Analytics

### 성능 요구사항

- **페이지 로딩 시간**: 2초 이하
- **API 응답 시간**: 1초 이하
- **에러율**: 1% 이하
- **가용성**: 99% 이상

### 보안 요구사항

## 4. 사용자 경험

### 주요 화면 플로우

#### 플로우 1: 편지 생성 및 전송

```
메인 화면
  ↓
"편지 만들기" 클릭
  ↓
곡 검색 및 추가 화면
  ↓
메시지 작성 화면
  ↓
미리보기 화면
  ↓
전송 완료 화면
```

#### 플로우 2: 편지 수신 및 저장

```
링크 클릭 (이메일/SNS)
  ↓
편지 열람 페이지 (로그인 불필요)
  ↓
"내 플랫폼에 저장" 클릭
  ↓
플랫폼 선택
  ↓
로그인 (처음 사용 시)
  ↓
저장 완료
```

#### 플로우 3: 편지 보관함

```
로그인
  ↓
메인 화면 → "받은 편지" 탭
  ↓
편지 목록
  ↓
편지 클릭
  ↓
편지 상세 보기
```

### 인터랙션 설계

#### 편지 생성
- **곡 검색**: 실시간 검색 (debounce 300ms)
- **곡 추가**: 클릭 한 번으로 추가
- **곡 삭제**: 스와이프 또는 삭제 버튼
- **순서 변경**: 드래그 앤 드롭

#### 메시지 작성
- **자동 저장**: 5초마다 자동 저장
- **AI 생성**: 버튼 클릭 후 3초 내 생성
- **미리보기**: 실시간 미리보기

#### 플랫폼 저장
- **원클릭**: 버튼 한 번으로 저장
- **진행 상태**: 로딩 인디케이터
- **성공/실패**: 명확한 피드백

### 에러 처리

#### 곡 검색 실패
- "검색 결과가 없습니다" 메시지
- 검색어 수정 제안
- 수동 입력 옵션

#### 플랫폼 저장 실패
- 에러 메시지 표시
- 재시도 버튼
- 대안 플랫폼 제안

#### 네트워크 오류
- 오프라인 상태 감지
- 재시도 버튼
- 로컬 저장 (선택적)
