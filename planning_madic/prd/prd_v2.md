## 1. 제품 개요

### 비전과 목표

**비전**: "병원의 시한폭탄을 제거하고, 환자 개인을 보는 정밀 의료를 실현한다"

**MVP 목표**: Real-time Infection Guard를 통해 병원 내 감염 확산을 0초로 줄이는 핵심 가치를 검증한다.

### 핵심 가치 제안

1. **실시간 감염 관리**: 검사 결과 보고 전 감염 알림으로 원내 감염 확산 차단
2. **의료 사고 예방**: 명확한 ROI 제공 (의료 사고 소송 비용 절감)
3. **웹 기반 접근성**: 설치 없이 바로 사용 가능한 구독 모델

### 타겟 사용자

#### 1차 타겟: 중소형 병원 (병상 100-500개)

- **페르소나**: 의료 사고 리스크를 줄이고 싶은 병원 관리자
- **니즈**:
  - 의료 사고 예방 및 소송 비용 절감
  - 병원 평가 인증 점수 향상
  - 정밀 의료 서비스 제공
- **사용 시나리오**: 감염 관리, 검사 결과 모니터링, 의료 사고 예방

#### 2차 타겟: 대학병원 및 대형 병원

- **페르소나**: 정밀 의료 인프라를 확장하고 싶은 병원 관리자
- **니즈**:
  - 고도화된 감염 관리 시스템
  - 개인 맞춤형 정밀 진단
  - 데이터 기반 의료 의사결정
- **사용 시나리오**: 정밀 진단, 감염 관리 고도화, 의료 품질 향상

## 2. 기능 명세

### 우선순위 정의

- **P0 (필수)**: MVP의 핵심 가치를 검증하기 위해 반드시 필요
- **P1 (중요)**: 사용자 경험을 크게 향상시키지만 MVP에서는 선택적
- **P2 (향후)**: Phase 2에서 추가

### P0: 핵심 기능

#### 2.1 사용자 인증 및 권한 관리

**기능 설명**:
- 의료진 로그인 및 역할 기반 접근 제어
- HIPAA 준수 보안 인프라

**상세 요구사항**:
- 이메일/소셜 로그인 지원 (Google, Microsoft)
- 역할 기반 접근 제어:
  - 임상병리사: 검체 접수, 검사 결과 입력
  - 의사: 검사 결과 확인, 승인
  - 간호사: 알림 확인, 환자 정보 조회
  - 관리자: 시스템 설정, 통계 조회
- 다중 인증 (MFA) 지원
- 세션 관리 및 자동 로그아웃

**사용자 시나리오**:
1. 사용자가 "로그인" 버튼 클릭
2. 이메일/소셜 로그인 선택
3. 인증 완료 후 역할에 따른 대시보드 표시
4. 세션 만료 시 자동 로그아웃

**사용자 변화**:
- **Before**: 복잡한 시스템 로그인, 역할별 접근 제어 어려움, 보안 우려
- **After**: 간편한 로그인, 역할별 맞춤 대시보드, HIPAA 준수 보안으로 신뢰 확보
- **변화 지표**: 로그인 완료 시간 30초 이내, 보안 사고 0건, 사용자 만족도 4.0/5.0 이상
- **변화의 의미**: 의료진이 시스템을 "신뢰할 수 있는 도구"로 인식하게 됨

**레버리지 포인트 활용**:
- HIPAA 준수 보안으로 병원 신뢰도 확보
- 간편한 로그인으로 사용자 진입 장벽 낮춤

**성공 기준**:
- 로그인 완료까지 30초 이내
- 보안 사고 0건
- HIPAA 준수 100%

---

#### 2.2 검체 접수 및 고위험군 알림

**기능 설명**:
- 검체 접수 화면에서 고위험군 환자 정보 팝업 표시
- EMR 연동 (수동 파일 업로드 - Phase 1)

**상세 요구사항**:

**2.2.1 검체 접수**
- 검체 정보 입력 (환자 ID, 검체 종류, 접수 시간)
- 바코드 스캔 지원
- 검체 상태 관리 (접수, 진행중, 완료)

**2.2.2 고위험군 알림**
- EMR 파일 업로드 (CSV, Excel)
- 고위험군 환자 정보 팝업:
  - 감염 이력 (VRE, CRE, MRSA 등)
  - 투석 여부
  - 격리 필요 여부
- 알림 확인 및 기록

**사용자 시나리오**:
1. 임상병리사가 검체 접수 화면 진입
2. 환자 ID 입력 또는 바코드 스캔
3. 고위험군 정보 팝업 표시
4. 알림 확인 및 검체 처리 주의사항 확인
5. 검체 접수 완료

**사용자 변화**:
- **Before**: 검체 접수 시 고위험군 정보를 별도 시스템에서 확인해야 함, 확인 누락 가능성, 검체 오염 사고 발생
- **After**: 검체 접수 시 자동으로 고위험군 정보 표시, 사전 예방 가능, 검체 오염 사고 0건
- **변화 지표**: 고위험군 알림 확인률 90% 이상, 검체 오염 사고 0건, 검체 접수 시간 20% 단축
- **변화의 의미**: 임상병리사가 검체 접수를 "안전 관리의 첫 단계"로 인식하게 됨

**레버리지 포인트 활용**:
- 실시간 감염 관리의 사전 예방 단계
- 검체 오염 사고 예방으로 의료 사고 비용 절감

**성공 기준**:
- 고위험군 알림 표시 시간 3초 이내
- 알림 확인률 90% 이상
- 검체 오염 사고 0건

---

#### 2.3 검사 결과 입력 및 감염 알림

**기능 설명**:
- 검사 결과 입력 및 감염균 감지 시 실시간 알림 전송

**상세 요구사항**:

**2.3.1 검사 결과 입력**
- 검사 결과 수기 입력
- 검사 항목 선택 (미생물 검사, PCR 등)
- 결과값 입력 (Positive/Negative, 수치 등)
- 임시 저장 및 중간 보고

**2.3.2 감염균 감지 및 알림**
- 감염균 감지 로직:
  - 법정 전염병 (결핵, 코로나 등)
  - 다제내성균 (VRE, CRE, MRSA 등)
- 실시간 알림 전송:
  - 감염관리실 대시보드
  - 해당 병동 간호 스테이션 대시보드
  - 담당 의사 (이메일, SMS)
- 알림 내용:
  - 환자 정보
  - 감염균 종류
  - 격리 필요 여부
  - 권장 조치사항

**사용자 시나리오**:
1. 임상병리사가 검사 결과 입력 화면 진입
2. 검사 항목 선택 및 결과 입력
3. 감염균 감지 시 자동 알림 전송
4. 알림 전송 확인 및 이력 기록
5. 검사 결과 승인 대기

**사용자 변화**:
- **Before**: 검사 결과 보고 후 의료진 확인까지 시차 발생 (평균 30분-1시간), 그 사이 감염 확산 위험, 의료 사고 발생 가능
- **After**: 검사 결과 입력 즉시 알림 전송, 0초 대응 가능, 감염 확산 차단, 의료 사고 예방
- **변화 지표**: 알림 응답 시간 5분 이내, 감염 사고 0건, 의료 사고 소송 비용 50% 절감
- **변화의 의미**: 의료진이 검사 결과를 "즉시 대응해야 하는 신호"로 인식하게 됨

**레버리지 포인트 활용**:
- **핵심 레버리지**: 실시간 알림으로 의료 사고 예방
- 마케팅 메시지의 핵심: "감염 확산을 0초로 줄입니다"
- 경쟁자 모방 어려움: 기술적 진입장벽

**성공 기준**:
- 알림 전송 시간 3초 이내
- 알림 응답 시간 5분 이내
- 감염 사고 0건

#### 2.4 실시간 알림 대시보드

**기능 설명**:
- 실시간 알림 모니터링 및 관리 대시보드

**상세 요구사항**:

**2.4.1 알림 대시보드**
- 실시간 알림 목록 표시
- 알림 우선순위 표시 (긴급, 일반)
- 알림 상태 관리 (미확인, 확인중, 완료)
- 알림 필터링 (감염균 종류, 병동, 날짜)

**2.4.2 알림 상세 정보**
- 환자 정보
- 감염균 종류 및 상세 정보
- 검사 결과 상세
- 권장 조치사항
- 알림 이력

**2.4.3 알림 확인 및 추적**
- 알림 확인 버튼
- 조치사항 입력
- 알림 완료 처리
- 알림 이력 조회

**사용자 시나리오**:
1. 감염관리실 직원이 대시보드 진입
2. 실시간 알림 목록 확인
3. 긴급 알림 클릭하여 상세 정보 확인
4. 권장 조치사항 확인 및 조치 입력
5. 알림 완료 처리

**사용자 변화**:
- **Before**: 알림을 이메일이나 전화로 받아 확인 지연 (평균 10-15분), 알림 누락 가능성, 대응 지연으로 감염 확산
- **After**: 실시간 대시보드에서 즉시 확인 및 대응 가능 (평균 1분 이내), 알림 누락 없음, 즉시 대응으로 감염 확산 차단
- **변화 지표**: 알림 확인 시간 1분 이내, 알림 응답 시간 5분 이내, 알림 누락률 0%
- **변화의 의미**: 감염관리실 직원이 알림을 "즉시 처리해야 하는 업무"로 인식하게 됨

**레버리지 포인트 활용**:
- 실시간 감염 관리의 핵심 인터페이스
- 알림 누락 방지로 의료 사고 예방

**성공 기준**:
- 알림 확인 시간 1분 이내
- 알림 응답 시간 5분 이내
- 알림 누락률 0%

#### 2.5 EMR 파일 업로드 (Phase 1)

**기능 설명**:
- EMR 데이터 수동 파일 업로드 (자동 연동은 Phase 2)

**상세 요구사항**:
- 파일 형식 지원 (CSV, Excel)
- 파일 업로드 및 파싱
- 환자 정보 매칭
- 업로드 이력 관리
- 에러 처리 및 재시도

**사용자 시나리오**:
1. 관리자가 EMR 데이터 업로드 화면 진입
2. 파일 선택 및 업로드
3. 파일 파싱 및 환자 정보 매칭
4. 업로드 완료 확인
5. 업로드 이력 조회

**사용자 변화**:
- **Before**: EMR 데이터를 수동으로 입력하거나 별도 시스템에서 확인해야 함, 데이터 불일치 가능성
- **After**: 파일 업로드로 간편하게 데이터 연동, 자동 매칭으로 데이터 정확도 향상
- **변화 지표**: 파일 업로드 성공률 95% 이상, 데이터 입력 시간 80% 단축

**성공 기준**:
- 파일 업로드 성공률 95% 이상
- 파일 파싱 시간 10초 이내

### 2.6 핵심 레버리지 포인트

#### 레버리지 포인트 1: 실시간 감염 관리 (가장 중요)

**왜 중요한가**:
- 이것이 플랫폼의 핵심 차별화 포인트
- 명확한 ROI 제공 (의료 사고 비용 절감)
- 병원의 생존 문제 해결

**활용 방법**:
- MVP에서 이 기능을 완벽하게 구현해야 함
- 마케팅 메시지의 핵심으로 활용: "감염 확산을 0초로 줄입니다"
- 파일럿 병원에서 이 가치를 검증

**기능 매핑**:
- **2.3 검사 결과 입력 및 감염 알림**: 핵심 구현 - 검사 결과 입력 즉시 알림 전송으로 0초 대응
- **2.4 실시간 알림 대시보드**: 실시간 모니터링 - 대시보드에서 즉시 확인 및 대응
- **2.2 고위험군 알림**: 사전 예방 - 검체 접수 시 고위험군 정보 표시로 사전 예방

**마케팅 전략**:
- 병원 관리자 대상: "의료 사고 소송 비용 절감" - 구체적 수치 제시 (50% 절감)
- 의료진 대상: "감염 확산 예방으로 안전한 업무 환경" - 즉시 대응 가능성 강조

---

#### 레버리지 포인트 2: 웹 기반 구독 모델

**왜 중요한가**:
- 낮은 진입 장벽 (설치 불필요)
- 빠른 도입 가능
- 확장성과 경제성

**활용 방법**:
- MVP에서 웹 기반 아키텍처 완벽 구현
- 구독 모델로 빠른 수익화
- 클라우드 기반으로 자동 스케일링

**기능 매핑**:
- **2.1 사용자 인증**: 웹 기반 인증 - 브라우저에서 즉시 로그인
- **전체 시스템**: 웹 브라우저에서 즉시 사용 가능 - 설치 불필요

**마케팅 전략**:
- "설치 없이 바로 사용하는 웹 기반 솔루션" - 도입 시간 단축 강조
- "초기 도입 비용 최소화" - 구독 모델의 경제성 강조

---

#### 레버리지 포인트 3: HIPAA 준수 보안

**왜 중요한가**:
- 의료 데이터 보안 필수
- 법적 리스크 방지
- 병원 신뢰도 확보

**활용 방법**:
- MVP에서 HIPAA 준수 완벽 구현
- 보안 감사 및 인증 획득

### 3.4 스케일링 전략 상세

#### 알림 시스템 부하 관리 전략

**문제**: 병원 수 증가 시 알림 전송량 급증, WebSocket 연결 수 증가, 알림 전송 지연 가능성

**전략**:

1. **알림 큐잉 시스템**
   - Redis Queue를 활용한 알림 큐 관리
   - 우선순위 기반 큐 (긴급 > 일반)
   - 병원별 큐 분리로 부하 분산
   - 예상 효과: 알림 전송 지연 50% 감소

2. **WebSocket 연결 관리**
   - 연결 풀링 (Connection Pooling)
   - 병원별 연결 제한 (최대 100개 동시 연결)
   - 연결 만료 및 재연결 메커니즘
   - 예상 효과: 서버 부하 40% 감소

3. **알림 채널 최적화**
   - 다중 채널 활용 (이메일, SMS, 대시보드)
   - 채널별 부하 분산
   - 실패 시 자동 재시도 (Exponential backoff)
   - 예상 효과: 알림 전송 성공률 99% 이상

4. **모니터링 및 알림**
   - 알림 시스템 부하 실시간 모니터링
   - 병원당 알림 전송량 추적
   - 부하 80% 도달 시 자동 스케일링 트리거
   - 대시보드: CloudWatch + Datadog

**목표**: 병원 100곳까지 알림 전송 시간 3초 이내 유지

---

#### 데이터베이스 최적화 전략

**문제**: 병원 수 증가 시 검체 접수 및 검사 결과 데이터 증가, 실시간 알림 조회 쿼리 증가, 데이터베이스 성능 저하

**전략**:

1. **인덱싱 전략**
   - 환자 ID (Primary Key)
   - 검체 접수 시간 (정렬용)
   - 감염균 종류 (필터링용)
   - 알림 상태 (조회용)
   - 예상 효과: 쿼리 속도 10배 향상

2. **쿼리 최적화**
   - N+1 쿼리 방지 (JOIN 사용)
   - 페이지네이션 (20개씩)
   - 필요한 컬럼만 SELECT
   - 예상 효과: 쿼리 시간 50% 감소

3. **읽기/쓰기 분리 (필요 시)**
   - 읽기 전용 복제본 사용
   - AWS RDS Read Replicas 활용
   - 예상 효과: 읽기 성능 2배 향상

4. **캐싱 전략**
   - Redis 캐싱
   - 환자 정보 캐싱 (10분 TTL)
   - 알림 목록 캐싱 (5분 TTL)
   - 예상 효과: 데이터베이스 부하 60% 감소

**목표**: 병원 100곳까지 쿼리 응답 시간 1초 이하 유지

---

#### 에러 핸들링 전략

**문제**: EMR 연동 실패, 네트워크 오류, 알림 전송 실패 등

**전략**:

1. **Exponential Backoff 패턴**
   - 첫 재시도: 1초 후
   - 두 번째 재시도: 2초 후
   - 세 번째 재시도: 4초 후
   - 최대 3회 재시도
   - 예상 효과: 일시적 오류 90% 해결

2. **Circuit Breaker 패턴**
   - 알림 전송 실패율 50% 초과 시 5분간 차단
   - 자동 복구 시도
   - 사용자에게 명확한 에러 메시지
   - 예상 효과: 연쇄 실패 방지

3. **재시도 로직 상세화**
   - 네트워크 오류: 즉시 재시도
   - 알림 전송 오류 (4xx): 재시도 안 함
   - 알림 전송 오류 (5xx): Exponential backoff로 재시도
   - EMR 연동 오류: 수동 재시도 옵션 제공

4. **사용자 피드백 메커니즘**
   - 명확한 에러 메시지
   - 재시도 버튼 제공
   - 대안 채널 제시 (이메일 → SMS)
   - 고객 지원 채널 안내

**목표**: 에러율 1% 이하 유지, 사용자 만족도 유지

## 10. 초기 병원 확보 전략

### 전략 개요

MVP 출시 후 3개월 내 파일럿 병원 3곳 확보 목표

### 전략 1: 파일럿 프로그램

**목표**: 파일럿 병원 3곳 확보

**실행 방법**:

1. **파일럿 병원 모집**
   - 중소형 병원 (병상 100-500개) 대상
   - 의료 사고 리스크가 높은 병원 우선
   - 온라인 모집 공고 및 직접 연락
   - 목표: 10곳 후보 중 3곳 선정

2. **파일럿 혜택 제공**
   - 50% 할인 (병원당 월 500,000원)
   - 무료 온보딩 및 교육
   - 우선 기술 지원
   - 성공 사례 마케팅 협조

3. **성공 사례 구축**
   - 파일럿 기간 중 성과 측정
   - 감염 사고 0건 달성
   - 사용자 만족도 4.0/5.0 이상
   - 성공 사례 문서화 및 마케팅

**예상 효과**: 파일럿 병원 3곳 → 정식 고객 3곳 전환

### 전략 2: 의료진 교육 및 지원

**목표**: 사용자 만족도 향상 및 바이럴 효과

**실행 방법**:

1. **온보딩 교육**
   - 시스템 사용법 교육 (2시간)
   - 역할별 맞춤 교육
   - 실습 및 Q&A 세션
   - 교육 자료 제공 (매뉴얼, 동영상)

2. **지속적인 지원**
   - 전담 CS 담당자 배정
   - 주간 체크인 미팅
   - 피드백 수집 및 개선
   - 기술 지원 (이메일, 전화)

3. **커뮤니티 구축**
   - 사용자 커뮤니티 운영 (온라인 포럼)
   - 베스트 프랙티스 공유
   - 정기 웨비나 개최
   - 사용자 간 네트워킹

**예상 효과**: 사용자 만족도 4.0/5.0 이상, 바이럴 효과로 추가 병원 확보

### 전략 3: 마케팅 채널

**목표**: 다양한 채널을 통한 병원 확보

**실행 방법**:

1. **온라인 마케팅**
   - 의료 기관 대상 디지털 마케팅
   - LinkedIn, 의료 전문 커뮤니티 광고
   - SEO 최적화 (의료 사고 예방, 감염 관리 키워드)
   - 예산: 1,000,000원/월 × 3개월 = 3,000,000원

2. **오프라인 마케팅**
   - 의료 기관 전시회 참가
   - 병원 관리자 대상 세미나 개최
   - 의료진 대상 웨비나 개최
   - 예산: 2,000,000원/회 × 2회 = 4,000,000원

3. **콘텐츠 마케팅**
   - 의료 사고 예방 관련 블로그 포스팅
   - 감염 관리 베스트 프랙티스 공유
   - 성공 사례 케이스 스터디
   - 예산: 무료 (내부 리소스 활용)

**총 예산**: 7,000,000원 (3개월)

**예상 효과**:
- 온라인 마케팅: 5곳 문의
- 오프라인 마케팅: 3곳 문의
- 콘텐츠 마케팅: 2곳 문의
- **총: 10곳 문의 → 3곳 파일럿 선정**
