## 사용자 경험 요구사항 분석

### 핵심 가치 제안의 UX 관점

PRD의 핵심 가치 제안을 사용자 경험 관점에서 분석했습니다:

1. **플랫폼 중립성**
   - **UX 목표**: 사용자가 자신의 플랫폼을 선택할 수 있는 자유
   - **디자인 기회**: 플랫폼 선택을 명확하고 직관적으로 표시
   - **페인포인트**: 현재는 플랫폼이 다르면 공유 불가
   - **해결책**: 모든 플랫폼을 동등하게 표시, 선택을 쉽게

2. **감성 전달**
   - **UX 목표**: 음악을 "선물"로 인식하게 만들기
   - **디자인 기회**: 편지 형태의 감성적 디자인
   - **페인포인트**: 단순 링크는 맥락 없음
   - **해결책**: 카드 형태의 편지 디자인, 메시지 강조

3. **영구 보관**
   - **UX 목표**: 받은 편지를 추억으로 인식
   - **디자인 기회**: 시간순 정렬, 검색 기능, 시각적 아카이브
   - **페인포인트**: 링크는 대화방에서 묻힘
   - **해결책**: 보관함 UI, 날짜별/사람별 정리

## 타겟 사용자 페르소나 분석

### 1차 타겟: 일반 팬 & 음악 애호가 (20-35세)

**사용자 특성**:
- 모바일 우선 사용
- 빠른 의사결정 선호
- 감성적 연결 중시
- 소셜 공유 활발

**UX 요구사항**:
- 모바일 최적화 필수
- 빠른 편지 생성 (3분 이내)
- 공유 기능 강조
- 감성적 디자인

**디자인 방향**:
- 모바일 우선 설계
- 단계별 가이드 (Progressive Disclosure)
- 공유 버튼 명확히 표시
- 따뜻하고 친근한 톤

### 2차 타겟: 아티스트 & 음악인

**사용자 특성**:
- 데스크톱도 사용
- 셋리스트 공유 중요
- 브랜드 아이덴티티 중시
- 팬과의 소통 중요

**UX 요구사항**:
- 데스크톱 최적화
- 셋리스트 공유 기능 강조
- 아티스트 프로필 (Phase 2)
- 공연/페스티벌 컨텍스트

**디자인 방향**:
- 데스크톱 레이아웃 지원
- 셋리스트 전용 뷰
- 아티스트 배지/인증 (Phase 2)
- 공연 느낌의 디자인

## 기술 아키텍처의 제약사항 이해

### Frontend 제약사항

**Next.js 14 App Router**:
- ✅ 서버 컴포넌트 활용 가능
- ✅ 자동 코드 스플리팅
- ⚠️ 클라이언트 컴포넌트는 명시적 선언 필요
- ⚠️ 서버/클라이언트 경계 명확히 이해 필요

**디자인 영향**:
- 서버 컴포넌트로 초기 로딩 최적화
- 클라이언트 컴포넌트는 인터랙션 필요한 곳만
- 스켈레톤 UI는 서버 컴포넌트에서 구현

**Tailwind CSS**:
- ✅ 유틸리티 클래스로 빠른 개발
- ✅ 일관된 디자인 시스템 구축 가능
- ⚠️ 커스텀 스타일은 config 파일 활용

**디자인 영향**:
- 디자인 시스템을 Tailwind 기반으로 구축
- 커스텀 컬러/스페이싱은 tailwind.config.js에 정의

**shadcn/ui**:
- ✅ 접근성 기본 제공
- ✅ 커스터마이징 가능
- ⚠️ 컴포넌트를 프로젝트로 복사 필요

**디자인 영향**:
- shadcn/ui 컴포넌트를 기반으로 확장
- 접근성은 기본 제공되므로 추가 작업 최소화

### Backend 제약사항

**Supabase**:
- ✅ 실시간 기능 활용 가능
- ✅ Row Level Security로 보안
- ⚠️ 쿼리 성능은 인덱싱에 의존

**디자인 영향**:
- 실시간 자동 저장 UX 구현 가능
- 보안은 백엔드에서 처리되므로 UX에 영향 없음

**API 응답 시간**:
- 목표: 1초 이하
- 페이지 로딩: 2초 이하

**디자인 영향**:
- 로딩 상태 디자인 필수
- 스켈레톤 UI로 인지된 성능 향상
- 최적화된 이미지 사용

## 사용자 플로우 최적화 기회

### 플로우 1: 사용자 가입 및 온보딩

**현재 플로우**:
1. 로그인 페이지 → 소셜 로그인 선택
2. OAuth 콜백 → 자동 가입
3. 온보딩 페이지 → 닉네임 입력
4. 메인 페이지

**최적화 기회**:
- ✅ 온보딩을 한 화면에서 완료 (닉네임 + 프로필 이미지)
- ✅ 진행 표시기로 현재 단계 명확히 표시
- ✅ "나중에" 옵션 제공 (프로필 이미지)
- ✅ 첫 편지 생성 가이드 제공

**디자인 제안**:
- 단계별 진행 표시기 (Step 1/2)
- 프로필 이미지는 드래그 앤 드롭
- 온보딩 완료 후 "첫 편지 만들기" CTA

---

### 플로우 2: 편지 생성

**현재 플로우**:
1. 편지 생성 페이지
2. 곡 검색 및 추가
3. 메시지 작성
4. 미리보기
5. 전송

**최적화 기회**:
- ✅ 단계별 가이드 (곡 추가 → 메시지 → 미리보기)
- ✅ 진행 표시기로 현재 단계 표시
- ✅ 자동 저장으로 데이터 손실 방지
- ✅ 실시간 미리보기

**디자인 제안**:
- 좌측: 편집 영역 (곡 목록, 메시지 입력)
- 우측: 실시간 미리보기 (데스크톱)
- 모바일: 탭으로 전환 (편집/미리보기)
- 진행 표시기: 상단에 표시

---

### 플로우 3: 편지 열람 및 플레이리스트 저장

**현재 플로우**:
1. 편지 링크 클릭
2. 편지 열람
3. 플랫폼 선택
4. 로그인 (필요 시)
5. 저장 완료

**최적화 기회**:
- ✅ 편지 열람 시 즉시 "저장하기" CTA 표시
- ✅ 플랫폼 선택을 명확하게 (아이콘 + 이름)
- ✅ 저장 진행 상태 표시
- ✅ 저장 완료 후 플랫폼 링크 제공

**디자인 제안**:
- 편지 카드 형태로 표시
- 플랫폼 선택: 큰 아이콘 버튼
- 저장 중: 진행 표시기
- 저장 완료: 성공 애니메이션 + 링크

---

### 플로우 4: 편지 보관함

**현재 플로우**:
1. 보관함 페이지
2. 편지 목록 표시
3. 검색/필터
4. 편지 상세

**최적화 기회**:
- ✅ 그리드/리스트 뷰 전환
- ✅ 날짜별/사람별 필터
- ✅ 검색 기능 강화
- ✅ 무한 스크롤 또는 페이지네이션

**디자인 제안**:
- 그리드 뷰: 카드 형태 (시각적)
- 리스트 뷰: 컴팩트한 형태 (정보 중심)
- 필터: 상단에 탭 형태
- 검색: 상단 검색 바

## 디자인 기회 발견

### 기회 1: 감성적 편지 디자인

**기회**:
- 편지를 실제 편지처럼 디자인
- 엽서/카드 형태의 시각적 디자인
- 애니메이션으로 열림 효과

**영향**:
- 사용자가 편지를 "선물"로 인식
- 공유 의도 증가
- 브랜드 아이덴티티 강화

---

### 기회 2: 플랫폼 중립성 시각화

**기회**:
- 모든 플랫폼을 동등하게 표시
- 플랫폼 아이콘을 명확하게
- "어떤 플랫폼이든" 메시지 강조

**영향**:
- 핵심 가치 제안 명확히 전달
- 사용자 혼란 감소
- 차별화 포인트 강조

---

### 기회 3: 모바일 우선 경험

**기회**:
- 모바일에서 완벽한 경험 제공
- 터치 인터랙션 최적화
- 스와이프 제스처 활용

**영향**:
- 타겟 사용자(20-35세) 만족도 증가
- 사용성 향상
- 바이럴 확산 용이

## 기술적 제약 내 UX 설계 방향

### 성능 최적화를 위한 디자인

**로딩 상태**:
- 스켈레톤 UI로 인지된 성능 향상
- 최소 200ms 표시로 깜빡임 방지
- 진행 표시기로 명확한 피드백

**이미지 최적화**:
- Next.js Image 컴포넌트 사용
- WebP 포맷
- 적절한 크기로 리사이징

**코드 스플리팅**:
- 라우트별 자동 스플리팅
- 큰 컴포넌트는 동적 임포트

---

### 접근성 고려사항

**WCAG 2.1 AA 준수**:
- 색상 대비 비율 4.5:1 이상
- 키보드 네비게이션 지원
- 스크린 리더 지원
- 포커스 인디케이터 명확히

**디자인 영향**:
- 컬러 팔레트 설계 시 대비 고려
- 인터랙션 요소는 키보드 접근 가능
- ARIA 레이블 적절히 사용
