## 핵심 디자인 결정

### 브랜드 아이덴티티

**톤 & 매너**:
- 따뜻하고 친근함
- 진정성 있고 솔직함
- 감성적이지만 과하지 않음

**키워드**:
- Connection (연결)
- Emotion (감성)
- Freedom (자유)
- Warmth (따뜻함)
- Music (음악)

---

### 컬러 시스템

**Primary**: Indigo (`#6366F1`)
- 신뢰, 창의성, 음악의 깊이

**Secondary**: Pink (`#EC4899`)
- 따뜻함, 감성, 연결

**Accent**: Amber (`#F59E0B`)
- 에너지, 주목, 음악의 리듬

**다크 모드**: 완전 지원

---

### 타이포그래피

**폰트**: Pretendard (한글 최적화)
- 한글 가독성 우수
- 웹폰트 지원
- 무료 사용 가능

**스케일**: 8단계 (H1 ~ Caption)
- 반응형 크기 조정
- 명확한 계층 구조

---

### 디자인 철학

1. **Form Follows Function**: 기능에서 아름다움
2. **Progressive Disclosure**: 점진적 정보 공개
3. **Consistent, Not Uniform**: 일관성 유지, 컨텍스트에 맞게 변주
4. **Accessible by Default**: 접근성은 필수

---

## 사용자 경험 최적화

### 플로우 최적화

**가입 플로우**:
- ✅ 진행 표시기로 현재 단계 명확히
- ✅ 한 화면에서 완료 (닉네임 + 프로필)
- ✅ "나중에" 옵션 제공

**편지 생성 플로우**:
- ✅ 단계별 가이드 (곡 추가 → 메시지 → 미리보기)

- ✅ 자동 저장으로 데이터 손실 방지

**편지 열람 플로우**:
- ✅ 즉시 "저장하기" CTA 표시
- ✅ 플랫폼 선택 명확하게
- ✅ 저장 진행 상태 표시

**네비게이션**:

## 디자인 트렌드 적용

### 적용된 트렌드

**Glassmorphism**:
- 편지 카드 배경에 선택적 적용
- 브랜드 아이덴티티와 조화롭게

**Bold Typography**:
- 히어로 섹션에 대담한 타이포그래피
- 감성적 메시지 강조

**Micro-interactions**:
- 버튼 클릭, 곡 추가 등
- 즉각적 피드백 제공

**Dark Mode**:
- 완전 지원
- 모든 컬러 다크 모드 팔레트 제공

---

### 적용하지 않은 트렌드

**Neumorphism**:
- 브랜드 아이덴티티와 맞지 않음
- 접근성 고려 (대비)

**3D Elements**:
- 성능 고려
- MVP 범위 밖

---

## PM/TPM 협업 포인트

### PM과의 협업

**사용자 경험 관점 제안**:
- 단계별 가이드로 사용자 혼란 감소
- 진행 표시기로 현재 위치 명확히
- 자동 저장으로 데이터 손실 방지

**플로우 최적화 제안**:
- 온보딩을 한 화면에서 완료
- 편지 생성 시 실시간 미리보기
- 편지 열람 시 즉시 저장 CTA

---

### TPM과의 협업

**기술적 제약 이해**:
- Next.js Server Components 활용
- Tailwind CSS 기반 디자인 시스템
- shadcn/ui 컴포넌트 확장

**성능과 UX의 균형**:
- 스켈레톤 UI로 로딩 최적화
- 코드 스플리팅 고려한 디자인
- 이미지 최적화 전략

**컴포넌트 재사용성**:
- 디자인 시스템으로 일관성 유지
- 재사용 가능한 컴포넌트 설계
- 확장 가능한 구조

## PM/TPM 검토 필요 사항

### 질문 1: 디자인 시스템 선택

선정한 컬러와 타이포그래피가 브랜드 아이덴티티와 일치하는지 검토 부탁드립니다.

**Designer 의견**: 음악의 감성과 에너지를 표현하면서도 사용자가 편안하게 느낄 수 있는 컬러 팔레트를 선택했습니다.

### 질문 2: 모바일 우선 설계

모바일 우선 설계가 타겟 사용자(20-35세)에게 적절한지 검토 부탁드립니다.

**Designer 의견**: 타겟 사용자는 모바일 우선 사용자이므로 모바일 경험을 최우선으로 설계했습니다.

### 질문 3: 접근성 수준

WCAG 2.1 AA 기준이 MVP에 적절한지 검토 부탁드립니다.

**Designer 의견**: 접근성은 선택이 아닌 필수입니다. MVP에서도 AA 기준을 준수해야 합니다.
