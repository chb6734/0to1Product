## 대화 시작

### 🚀 CEO: PRD v3 검토 시작

안녕하세요, PM님, TPM님. PRD v3와 사용자 시나리오를 검토하겠습니다.

먼저 핵심 질문부터 시작하겠습니다:

**"이 PRD가 사용자를 유입시킬 수 있는가?"**

현재 PRD를 보니 7개 화면이 모두 구현되어 있고, 기본 플로우는 갖춰져 있습니다. 하지만 몇 가지 의문이 듭니다:

1. **랜딩 페이지의 차별점이 명확한가?**
   - "플랫폼 중립적 음악 공유"라는 메시지가 충분히 강력한가?
   - 사용자가 "아, 이건 내가 원하던 거구나"라고 즉시 이해할 수 있는가?

2. **첫 사용자 경험이 충분히 매끄러운가?**
   - 로그인 → 온보딩 → 편지 생성까지의 플로우가 번거롭지 않은가?

3. **바이럴 요소가 있는가?**
   - 사용자가 편지를 공유하고 싶게 만드는 요소가 있는가?

### 🧭 PM: CEO 질문에 대한 답변

CEO님의 질문에 답변드리겠습니다:

**1. 차별점 명확성:**
현재 PRD에서는 "플랫폼 중립성"을 강조하고 있지만, 실제 사용자 관점에서는 "친구가 Spotify 쓰는데 나는 Apple Music 써서 음악 공유가 안 되는 문제"를 해결한다는 메시지가 더 명확할 수 있습니다.

**2. 첫 사용자 경험:**
현재 플로우는:
- 랜딩 → 로그인 → 온보딩 → 편지 생성
- 총 4단계로, 각 단계마다 사용자가 이탈할 수 있는 포인트가 있습니다.

**3. 바이럴 요소:**
현재는 링크 공유와 QR 코드만 있습니다. 더 강력한 바이럴 요소가 필요할 수 있습니다.

### ⚙️ TPM: 기술적 관점에서의 검토

CEO님, PM님. 기술적 관점에서 몇 가지 우려사항을 제시하겠습니다:

**1. 곡 검색 기능의 기술적 복잡도:**
- PRD에서 "곡 검색"을 언급하고 있지만, 어떤 API를 사용할지 명확하지 않습니다.
- Spotify, Apple Music, YouTube Music 각각 다른 API를 사용해야 하는데, 통합 검색은 기술적으로 복잡합니다.
- 사용자가 직접 입력하는 방식도 고려해야 할 것 같습니다.

**2. 플랫폼 연동의 사용자 경험:**
- 편지를 받은 사용자가 "내 플랫폼에 저장"하려면 각 플랫폼에 로그인해야 합니다.
- 이 과정이 번거롭다면 사용자가 포기할 수 있습니다.

**3. 성능 및 확장성:**
- 편지 상세 페이지에서 플레이리스트를 표시할 때, 각 곡의 플랫폼별 링크를 실시간으로 생성해야 합니다.
- 이는 API 호출이 많아질 수 있고, 성능 문제가 발생할 수 있습니다.

## 구체적 개선안 논의

### 🚀 CEO: 데모 모드 상세 제안

데모 모드를 P1이 아닌 **P0으로 올려야 합니다.** 사용자 유입이 가장 중요한 MVP 목표이므로, 로그인 장벽을 낮추는 것이 필수입니다.

**데모 모드 상세 명세:**

**1. 데모 모드 진입:**
- 랜딩 페이지에서 "데모 보기" 또는 "로그인 없이 체험하기" 버튼 클릭
- 즉시 편지 생성 페이지로 이동 (데모 모드)
- 로컬 스토리지에 `demo_mode: true` 플래그 저장

**2. 데모 모드 제한사항:**
- 편지 저장 불가 (로컬 스토리지에만 임시 저장)
- 링크 공유 불가 (또는 임시 링크만 생성, 24시간 후 만료)
- 편지 수정 가능 (로컬 스토리지 내에서만)

**3. 데모 모드에서 로그인 전환:**
- "저장하려면 로그인이 필요합니다" 메시지 표시
- 로그인 완료 후 로컬 스토리지 데이터를 서버로 마이그레이션
- 사용자가 만든 편지가 그대로 보관함에 저장됨

**이렇게 하면 사용자는 서비스를 먼저 체험하고, 만족하면 로그인하게 됩니다.**

### ⚙️ TPM: 데모 모드 기술 명세

PM님의 UX 설계를 기술적으로 검토했습니다:

- 브라우저 종료 시에도 유지 (로컬 스토리지 특성)
- 용량 제한: 약 5-10MB (충분함)

**3. 로그인 후 마이그레이션:**
```typescript
// 로그인 완료 후
const demoData = localStorage.getItem('demo_letter_data');
if (demoData) {
  // 서버로 마이그레이션
  await migrateDemoLetter(JSON.parse(demoData));
  localStorage.removeItem('demo_mode');
  localStorage.removeItem('demo_letter_data');
}
```

**4. 임시 링크 생성:**
- 서버리스 함수로 임시 링크 생성
- 24시간 TTL 설정
- 만료 후 자동 삭제

**기술적으로 실현 가능하며, 복잡도도 낮습니다.**
