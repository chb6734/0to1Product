## 대화 시작

### 🧭 PM: 화면 분석 시작

안녕하세요, Designer님. 현재 구현된 화면들을 확인했습니다. 총 7개의 주요 화면이 있네요:

1. **랜딩 페이지** (`/`)
2. **로그인 페이지** (`/login`)
3. **온보딩 페이지** (`/onboarding`)
4. **편지 생성 페이지** (`/create`)
5. **보관함 페이지** (`/inbox`)
6. **둘러보기 페이지** (`/discover`)
7. **편지 상세 페이지** (`/letters/[id]`)

각 화면에서 사용자가 어떤 동작을 해야 하는지, 그리고 사용자 경험 관점에서 개선할 점이 있는지 함께 분석해보겠습니다.

### 🎨 Designer: 사용자 중심 분석 제안

좋습니다, PM님. 사용자 경험 관점에서 각 화면을 분석하겠습니다.

먼저 **사용자 여정(User Journey)**을 정의해야 할 것 같습니다:

1. **신규 사용자 여정**: 랜딩 → 로그인 → 온보딩 → 편지 생성 → 편지 공유
2. **기존 사용자 여정**: 로그인 → 보관함/둘러보기 → 편지 상세 → 편지 생성

각 화면에서 사용자의 **목표**와 **행동**을 명확히 정의하고, **감정적 여정**도 고려해야 합니다.

## 화면별 분석 및 PRD 작성

각 화면을 하나씩 분석하면서 PRD를 작성하겠습니다.

---

## 1. 랜딩 페이지 (`/`) 분석

### 🧭 PM: 랜딩 페이지 목적 정의

랜딩 페이지는 서비스의 첫인상을 결정하는 중요한 화면입니다. 현재 구현을 보니:

**현재 구현된 기능:**
- 히어로 섹션 (서비스 소개)
- 추천 편지 섹션 (3개 카드)
- 이용 방법 섹션 (3단계)
- CTA 섹션 (가입 유도)

**사용자 목표:**
1. 서비스가 무엇인지 빠르게 이해
2. 서비스의 가치를 확인
3. 다음 액션(가입/편지 만들기)을 취하고 싶게 만들기

### 🎨 Designer: 사용자 경험 관점 분석

좋습니다, PM님. 사용자 경험 관점에서 몇 가지 개선점을 제안합니다:

**현재 상태 분석:**
- ✅ 히어로 섹션이 명확함 ("어떤 플랫폼이든 음악을 공유하세요")
- ✅ CTA 버튼이 눈에 띔 (노란색 "편지 만들기")
- ⚠️ 추천 편지 섹션이 정적임 (실제 데이터가 아님)
- ⚠️ 데모 보기 버튼이 동작하지 않음

**개선 제안:**
1. **데모 보기 기능**: 실제 편지 상세 페이지로 이동하거나 비디오 데모 제공
2. **추천 편지 동적 로딩**: 실제 공개된 편지 데이터를 표시
3. **스크롤 애니메이션**: 섹션별 등장 애니메이션으로 몰입감 향상
4. **소셜 증명**: 사용자 수, 편지 수 등 통계 표시

**사용자 행동 플로우:**
```
랜딩 페이지 진입
  ↓
히어로 섹션 스크롤
  ↓
추천 편지 클릭 → 편지 상세 페이지
  또는
"편지 만들기" 클릭 → 로그인/편지 생성 페이지
  또는
"데모 보기" 클릭 → 데모 화면
```

### 🧭 PM: 동작 명세 정리

네, Designer님의 제안을 반영하여 동작 명세를 정리하겠습니다:

**필수 동작 (P0):**
1. "편지 만들기" 버튼 클릭 → 로그인 상태 확인 → 로그인 페이지 또는 편지 생성 페이지로 이동
2. 추천 편지 카드 클릭 → 편지 상세 페이지로 이동
3. "전체 보기" 클릭 → 둘러보기 페이지로 이동
4. "첫 편지 만들기" CTA 클릭 → 로그인 상태 확인 → 로그인 페이지 또는 편지 생성 페이지로 이동

**개선 동작 (P1):**
1. "데모 보기" 버튼 클릭 → 데모 모달 또는 데모 페이지 표시
2. 스크롤 시 섹션별 페이드인 애니메이션
3. 추천 편지 섹션에 실제 공개 편지 데이터 표시

## 2. 로그인 페이지 (`/login`) 분석

### 🎨 Designer: 로그인 UX 분석

로그인 페이지는 사용자가 서비스를 시작하는 첫 단계입니다. 현재 구현을 보니:

**현재 구현:**
- 3가지 소셜 로그인 옵션 (Google, Kakao, Apple)
- 약관 동의 텍스트
- 깔끔한 중앙 정렬 레이아웃

**사용자 경험 관점:**
- ✅ 소셜 로그인으로 가입 장벽이 낮음
- ✅ 약관 동의가 명확함
- ⚠️ 로그인 실패 시 에러 처리 필요
- ⚠️ 로딩 상태 표시 필요

**개선 제안:**
1. 로그인 버튼 클릭 시 로딩 스피너 표시
2. 로그인 실패 시 명확한 에러 메시지
3. 이미 로그인된 사용자는 자동으로 메인 페이지로 리다이렉트

### 🧭 PM: 로그인 플로우 명세

**정상 플로우:**
```
로그인 페이지 진입
  ↓
소셜 로그인 선택 (Google/Kakao/Apple)
  ↓
소셜 인증 완료
  ↓
신규 사용자 → 온보딩 페이지
기존 사용자 → 메인 페이지 (보관함)
```

**에러 케이스:**
1. 네트워크 오류 → "연결에 실패했습니다. 다시 시도해주세요."
2. 인증 거부 → "로그인이 취소되었습니다."
3. 서버 오류 → "일시적인 오류가 발생했습니다. 잠시 후 다시 시도해주세요."

## 3. 온보딩 페이지 (`/onboarding`) 분석

### 🎨 Designer: 온보딩 UX 분석

온보딩은 사용자가 서비스를 처음 사용할 때 설정하는 중요한 단계입니다.

**현재 구현:**
- 닉네임 입력 (필수, 최대 20자)
- 프로필 이미지 업로드 (선택)
- "시작하기" 버튼

**사용자 경험 관점:**
- ✅ 닉네임 입력이 간단함
- ✅ 프로필 이미지가 선택 사항임 (진입 장벽 낮춤)
- ⚠️ 닉네임 중복 검사 필요
- ⚠️ 프로필 이미지 업로드 기능 미구현
- ⚠️ 닉네임 유효성 검사 필요 (특수문자, 공백 등)

**개선 제안:**
1. 닉네임 실시간 중복 검사
2. 닉네임 유효성 검사 (한글/영문/숫자만 허용, 최소 2자)
3. 프로필 이미지 업로드 기능 구현
4. 기본 아바타 생성 (이니셜 기반)

### 🧭 PM: 온보딩 동작 명세

**정상 플로우:**
```
온보딩 페이지 진입
  ↓
닉네임 입력 (실시간 검증)
  ↓
프로필 이미지 선택 (선택적)
  ↓
"시작하기" 클릭
  ↓
프로필 저장
  ↓
메인 페이지로 이동
```

**검증 규칙:**
- 닉네임: 2-20자, 한글/영문/숫자만 허용, 중복 불가
- 프로필 이미지: 최대 5MB, JPG/PNG/GIF 형식

## 4. 편지 생성 페이지 (`/create`) 분석

### 🎨 Designer: 편지 생성 UX 분석

편지 생성은 서비스의 핵심 기능입니다. 현재 2단계 프로세스로 구현되어 있네요.

**현재 구현:**
- Step 1: 곡 추가, 메시지 작성, 공개 설정
- Step 2: 완성 화면 (링크 공유, QR 코드)

**사용자 경험 관점:**
- ✅ 단계별 프로세스로 명확함
- ✅ 실시간 미리보기 제공
- ⚠️ 곡 검색 기능 미구현
- ⚠️ 곡 추가/삭제 기능 미구현
- ⚠️ 메시지 템플릿 기능 미구현

**개선 제안:**
1. 곡 검색 시 자동완성 및 실시간 결과 표시
2. 추가된 곡 목록 표시 및 순서 변경 기능
3. 메시지 템플릿 제공 (생일, 위로, 응원 등)
4. 진행 표시기 (Step 1/2) 추가
5. 저장 기능 (임시 저장)

### 🧭 PM: 편지 생성 동작 명세

**Step 1 동작:**
1. 곡 검색 입력 → 자동완성 결과 표시
2. 곡 선택 → 플레이리스트에 추가
3. 곡 삭제 → 플레이리스트에서 제거
4. 곡 순서 변경 → 드래그 앤 드롭
5. 메시지 작성 → 실시간 미리보기 업데이트
6. 템플릿 선택 → 메시지 자동 입력
7. 공개 설정 변경 → 비공개/공개 토글
8. "다음" 클릭 → Step 2로 이동 (메시지 필수)

**Step 2 동작:**
1. 링크 복사 → 클립보드에 복사 및 토스트 메시지
2. QR 코드 표시 → 편지 링크 포함
3. "수정하기" 클릭 → Step 1로 돌아가기
4. "완료" 클릭 → 편지 저장 및 보관함으로 이동

## 5. 보관함 페이지 (`/inbox`) 분석

### 🎨 Designer: 보관함 UX 분석

보관함은 사용자가 받은 편지와 보낸 편지를 관리하는 공간입니다.

**현재 구현:**
- 탭 전환 (받은 편지 / 보낸 편지)
- 편지 카드 그리드 레이아웃
- 빈 상태 처리

**사용자 경험 관점:**
- ✅ 탭 전환이 직관적임
- ✅ 편지 카드가 정보를 잘 전달함
- ⚠️ 필터/정렬 기능 없음
- ⚠️ 검색 기능 없음
- ⚠️ 무한 스크롤 또는 페이지네이션 없음

**개선 제안:**
1. 정렬 옵션 (최신순, 인기순, 곡 수순)
2. 검색 기능 (보낸 사람, 메시지 내용)
3. 무한 스크롤 또는 페이지네이션
4. 편지 삭제 기능
5. 편지 즐겨찾기 기능

### 🧭 PM: 보관함 동작 명세

**기본 동작:**
1. 탭 전환 → 받은 편지/보낸 편지 목록 변경
2. 편지 카드 클릭 → 편지 상세 페이지로 이동
3. 빈 상태 표시 → "편지 만들기" CTA 제공

**향후 기능 (P1):**
1. 정렬 변경 → 목록 재정렬
2. 검색 입력 → 필터링된 결과 표시
3. 스크롤 → 추가 편지 로드 (무한 스크롤)
4. 편지 삭제 → 확인 모달 후 삭제

## 6. 둘러보기 페이지 (`/discover`) 분석

### 🎨 Designer: 둘러보기 UX 분석

둘러보기는 공개된 편지를 탐색하는 공간입니다.

**현재 구현:**
- 편지 카드 그리드 레이아웃
- 기본 목록 표시

**사용자 경험 관점:**
- ✅ 간단한 레이아웃
- ⚠️ 필터/카테고리 없음
- ⚠️ 검색 기능 없음
- ⚠️ 정렬 옵션 없음
- ⚠️ 인기 편지/최신 편지 구분 없음

**개선 제안:**
1. 카테고리 필터 (장르, 분위기 등)
2. 검색 기능
3. 정렬 옵션 (인기순, 최신순)
4. 추천 편지 섹션
5. 트렌딩 편지 섹션

### 🧭 PM: 둘러보기 동작 명세

**기본 동작:**
1. 편지 카드 클릭 → 편지 상세 페이지로 이동
2. 스크롤 → 추가 편지 로드

**향후 기능 (P1):**
1. 카테고리 선택 → 필터링된 결과 표시
2. 검색 입력 → 검색 결과 표시
3. 정렬 변경 → 목록 재정렬

## 7. 편지 상세 페이지 (`/letters/[id]`) 분석

### 🎨 Designer: 편지 상세 UX 분석

편지 상세 페이지는 편지의 전체 내용을 보여주는 핵심 화면입니다.

**현재 구현:**
- 보낸 사람 정보
- 메시지 표시
- 통계 (곡 수, 재생 수, 좋아요 수)
- 플레이리스트 표시
- 전체 재생 버튼

**사용자 경험 관점:**
- ✅ 정보 구조가 명확함
- ✅ 플레이리스트가 잘 표시됨
- ⚠️ 좋아요 기능 미구현
- ⚠️ 공유 기능 미구현
- ⚠️ 곡 재생 기능 미구현
- ⚠️ 플랫폼별 링크 없음

**개선 제안:**
1. 좋아요 버튼 동작 구현
2. 공유 버튼 (링크 복사, SNS 공유)
3. 곡별 플랫폼 링크 표시 (Spotify, Apple Music 등)
4. 곡 재생 미리보기 (YouTube API 활용)
5. 댓글 기능 (향후)

### 🧭 PM: 편지 상세 동작 명세

**기본 동작:**
1. 뒤로 가기 → 이전 페이지로 이동
2. 좋아요 클릭 → 좋아요 토글 및 카운트 업데이트
3. 전체 재생 클릭 → 플레이리스트 재생 (플랫폼 연동)
4. 곡 클릭 → 해당 플랫폼에서 재생
5. 공유 버튼 → 링크 복사 또는 SNS 공유

**향후 기능 (P1):**
1. 댓글 작성 → 댓글 추가
2. 편지 신고 → 신고 모달 표시
3. 편지 삭제 (본인이 보낸 편지인 경우)

## 사용자 플로우 다이어그램

다음은 주요 사용자 플로우를 다이어그램으로 정리했습니다.

### 신규 사용자 플로우

```
랜딩 페이지 (/)
  ↓ [편지 만들기 클릭]
로그인 페이지 (/login)
  ↓ [소셜 로그인]
온보딩 페이지 (/onboarding)
  ↓ [닉네임 입력 및 시작하기]
편지 생성 페이지 Step 1 (/create)
  ↓ [곡 추가 및 메시지 작성]
편지 생성 페이지 Step 2 (/create?step=2)
  ↓ [완료 클릭]
보관함 페이지 (/inbox)
```

### 기존 사용자 플로우

```
랜딩 페이지 (/)
  ↓ [로그인 클릭]
로그인 페이지 (/login)
```
